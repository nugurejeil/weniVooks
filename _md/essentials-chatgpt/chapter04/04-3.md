## 4.1 생성형 AI로 데이터 분석하기

이제 수집한 데이터를 ChatGPT, Claude와 같은 생성형 AI로 분석해봅시다.

### 4.1.1 왜 생성형 AI인가?

**전통적인 방법:**
```
데이터 → Excel 수식 → 차트 → 해석
또는
데이터 → Python 코딩 → 통계 분석 → 시각화 → 해석
```

**생성형 AI 방법:**
```
데이터 + 질문 → AI → 분석 + 시각화 + 인사이트
```

**장점:**
```
✅ 코딩 불필요
✅ 통계 지식 최소화
✅ 빠른 프로토타이핑
✅ 다양한 관점 제시
✅ 자연어로 소통

⚠️ 주의사항:
- Hallucination (잘못된 정보)
- 결과 검증 필수
- 중요한 의사결정은 전문가와 협업
```

```

총 347개의 관광지 정보:
- 컬럼: 이름, 주소, 카테고리, 평점, 리뷰수, 입장료
- 결측치: 입장료 23개 (무료인 경우)
- 평점 범위: 3.2 ~ 4.8 (평균 4.1)

주요 발견:
```

| 카테고리 | 관광지 수 | 평균 평점 | 평균 리뷰수 |

**인사이트:**
- 자연 관광지가 평점, 리뷰수 모두 1위
- 제주 = 자연 경관의 이미지와 일치
- 레저, 문화 카테고리는 개선 여지

### 4.2.3 실전 분석 2: 시각화

**프롬프트:**
```
"다음 시각화를 만들어줘:

1. 카테고리별 관광지 수 (막대 그래프)
2. 평점 분포 (히스토그램)
3. 입장료 vs 평점 (산점도)
4. TOP 10 인기 관광지 (가로 막대 그래프)

한글 폰트 사용하고, 색상은 제주의 느낌(바다, 자연)으로 해줘."
```

**AI가 생성하는 시각화:**

```python
import matplotlib.pyplot as plt
import pandas as pd

# 한글 폰트 설정
plt.rcParams['font.family'] = 'Malgun Gothic'
plt.rcParams['axes.unicode_minus'] = False

# 제주 컬러 팔레트 (바다, 감귤, 하늘)
colors = ['#0077be', '#ff8c00', '#87ceeb', '#98d8c8']

fig, axes = plt.subplots(2, 2, figsize=(16, 12))

# 1. 카테고리별 관광지 수
category_counts = df['카테고리'].value_counts()

               color=colors[0])

# 2. 평점 분포
axes[0, 1].hist(df['평점'], bins=20, color=colors[1],
                edgecolor='black')
axes[0, 1].set_title('평점 분포', fontsize=16)
axes[0, 1].set_xlabel('평점')
axes[0, 1].set_ylabel('빈도')

# 3. 입장료 vs 평점
axes[1, 0].scatter(df['입장료'], df['평점'],
                   alpha=0.6, color=colors[2])
axes[1, 0].set_title('입장료와 평점의 관계', fontsize=16)
axes[1, 0].set_xlabel('입장료 (원)')
axes[1, 0].set_ylabel('평점')

# 4. TOP 10 인기 관광지
top10 = df.nlargest(10, '리뷰수')[['이름', '리뷰수']]

                color=colors[3])
axes[1, 1].set_title('TOP 10 인기 관광지', fontsize=16)
axes[1, 1].set_xlabel('리뷰 수')

plt.tight_layout()
plt.savefig('제주관광지_분석.png', dpi=300, bbox_inches='tight')
```

### 4.2.4 실전 분석 3: 인사이트 도출

**프롬프트:**
```
"이 데이터에서 비즈니스 인사이트를 도출해줘.

3. 관광객: 어떤 관광지를 선택할까?

각 관점에서 데이터 기반으로 3가지씩 제안해줘."
```

**AI 응답 예시:**

**1. 신규 관광사업자**

```
📍 위치 선정:
- 동부 지역: 관광지는 많지만 숙박/음식점 부족
- 데이터: 동부 관광지 120개 vs 숙박 45개
- 제안: 동부 지역 펜션/게스트하우스 유망

🎯 콘셉트 선정:
- 문화/레저 카테고리는 평점 낮음 (3.9점)
- 제안: 차별화된 문화 체험 프로그램
- 예: 제주 전통문화 + 현대적 감각 결합

💰 가격 전략:
- 무료 관광지 평점: 4.2점
- 유료 관광지 평점: 4.0점
- 제안: 합리적 가격 + 품질로 차별화
  (5,000~10,000원대, 평점 4.5+ 목표)
```

**2. 기존 관광지 운영자**

```
⭐ 평점 개선:
- 평점 4.0 미만 관광지: 38%
- 공통 불만: 시설 노후화, 안내 부족
- 제안: 시설 현대화 + 다국어 안내판

📱 온라인 평판 관리:
- 리뷰 1000개 이상: 8% (28개)
- 나머지는 온라인 노출 부족
- 제안: SNS 마케팅, 블로거 협업

🎁 부가 서비스:
- TOP 10 관광지 공통점: 포토존, 카페
- 제안: 인스타그램 감성 포토존 설치
```

**3. 관광객**

```
🌟 숨은 명소:
- 평점 4.5+ 이지만 리뷰 300개 미만
- 예: [AI가 구체적 명소 10개 제시]
- 장점: 덜 붐빔, 높은 만족도

💵 가성비 코스:
- 무료 관광지 중 평점 4.3+
- 1일 코스: 성산일출봉 → 섭지코지 → 광치기해변
- 총 비용: 0원, 예상 만족도: 매우 높음

📅 시기 선택:
- 비수기(11-2월): 평균 30% 저렴
- 방문객 수: 성수기 대비 60% 수준
- 제안: 비수기 방문으로 혼잡 피하기
```

## 4.3 Claude Projects 활용

### 4.3.1 Claude Projects란?

Claude Pro에서 제공하는 기능으로, 대화와 파일을 프로젝트 단위로 관리합니다.

**특징:**
```
✅ 여러 파일 동시 업로드 (최대 5개)
✅ 긴 문서 처리 (각 100K+ 토큰)
✅ 대화 히스토리 유지
✅ 팀 협업 가능
```

### 4.3.2 실전 활용: 종합 분석

**Step 1: 프로젝트 생성**

```
프로젝트명: "제주 관광 데이터 분석"

업로드 파일:
1. 제주관광지.csv
2. 제주숙박.csv
3. 제주방문객통계.csv
4. 제주음식점.csv
5. 분석가이드.txt (분석 방향 정리)
```

**Step 2: 커스텀 인스트럭션**

```

분석 원칙:
1. 항상 데이터 기반으로 답변
2. 통계적 근거 제시 (평균, 비율 등)
3. 실행 가능한 제안
4. 시각화 포함

출력 형식:
- 요약 (3줄 이내)
- 상세 분석
- 시각화
- 실행 제안
```

**Step 3: 복합 질문**

```
프롬프트:
"제주 관광 종합 분석 보고서를 작성해줘.

포함 내용:
1. 현황 분석
   - 관광지, 숙박, 음식점 지역별 분포
   - 방문객 추이 (2020-2024)

2. 문제점 발견
   - 편중 현상 (특정 지역 집중)
   - 성수기 과밀, 비수기 한산
   - 평점 낮은 지역/카테고리

3. 개선 방안
   - 지역별 특화 전략
   - 비수기 활성화 방안
   - 외국인 유치 전략

형식: 경영진 보고용 (5페이지 분량)
시각화: 5개 이상 포함"
```

## 4.5 단계별 분석 프로세스

### 4.5.1 1단계: 탐색적 데이터 분석 (EDA)

**목표:** 데이터의 전체적인 모습 파악

**프롬프트 템플릿:**

```
"[데이터명] 데이터의 탐색적 분석을 해줘.

1. 데이터 개요
   - 행/열 개수
   - 컬럼별 데이터 타입
   - 결측치 현황

2. 기술 통계
   - 숫자형 변수: 평균, 중앙값, 표준편차, 최소/최대
   - 범주형 변수: 고유값 개수, 최빈값

3. 분포 확인
   - 숫자형 변수 히스토그램
   - 범주형 변수 막대그래프

4. 이상치 탐지
   - Box plot
   - 이상치 개수 및 리스트

5. 상관관계
   - 숫자형 변수 간 상관계수
   - Heatmap 시각화

표와 그래프로 시각적으로 제시해줘."
```

### 4.5.2 2단계: 세부 분석

**목표:** 구체적인 질문에 답하기

**분석 유형별 프롬프트:**

**비교 분석:**
```
"[A]와 [B]를 비교 분석해줘.

비교 항목:
- 평균, 중앙값
- 분포 (히스토그램 겹쳐서)
- 통계적 유의성 (t-test)

어느 쪽이 더 우수하고, 차이가 통계적으로 유의한지 알려줘."
```

**시계열 분석:**
```
"방문객 데이터의 시계열 분석을 해줘.

1. 추세 (Trend): 장기적 증가/감소
2. 계절성 (Seasonality): 월별 패턴
3. 예측: 향후 6개월 방문객 수 예측
   (Moving Average 또는 간단한 회귀)

꺾은선 그래프로 시각화하고, 인사이트를 제시해줘."
```

**세분화 분석 (Segmentation):**
```
"관광지를 다음 기준으로 세분화해줘:

기준:
- 평점 (상/중/하)
- 리뷰수 (많음/보통/적음)

총 9개 그룹으로 나누고:
1. 각 그룹의 특징
2. 크기 (개수, 비율)
3. 전략적 우선순위

2x2 또는 3x3 매트릭스로 시각화해줘."
```

### 4.5.3 3단계: 인사이트 도출

**프롬프트:**

```
"지금까지의 분석 결과를 바탕으로:

1. 주요 발견 3가지 (데이터 기반)
2. 놀라운 점 (예상 밖의 패턴)
3. 우려사항 (문제점)
4. 기회 요소 (활용 가능한 강점)

각 항목마다:
- 구체적 수치 제시
- 비즈니스 임팩트 설명
- 실행 가능한 제안 1가지

경영진이 이해하기 쉽게 작성해줘."
```

### 4.5.4 4단계: 보고서 작성

**프롬프트:**


## 4.6 실전 사례: 전체 프로세스

### 4.6.1 배경

관광사업자가 제주에 새로운 펜션을 오픈하려고 합니다.

**질문:**
"어느 지역에 어떤 콘셉트로 오픈하면 좋을까?"

### 4.6.2 분석 과정

**1단계: 데이터 수집**

```
수집 데이터:
- 제주 숙박업소 현황 (위치, 가격, 평점)
- 제주 관광지 분포
- 지역별 방문객 통계
- 경쟁사 분석 (리뷰, 가격대)
```

**2단계: ChatGPT 분석**

**프롬프트 1:**
```
"제주에 펜션을 오픈하려고 해.

분석 데이터:
[숙박업소 데이터 업로드]
[관광지 데이터 업로드]

질문:
1. 어느 지역이 숙박업소 대비 관광지가 많나? (공급 부족 지역)
2. 지역별 평균 숙박비는?
3. 평점 4.5+ 펜션의 공통점은?

표와 그래프로 보여줘."
```

**AI 응답:**

**공급 부족 지역:**

| 지역 | 관광지 수 | 숙박업소 수 | 비율 | 기회 |

## 4.12 NotebookLM 활용

### 4.12.1 NotebookLM이란?

Google의 AI 노트 도구로, 여러 문서를 업로드하고 통합 분석할 수 있습니다.

**특징:**
```
✅ 최대 50개 문서 업로드
✅ PDF, 웹 페이지, 텍스트 지원
✅ 출처 표시 (Hallucination 감소)
✅ 팟캐스트 생성 (AI 음성)
```

### 4.12.2 실전 활용

**Step 1: 소스 업로드**

```
업로드:
1. 제주관광공사 연례보고서 (PDF)
2. 공공데이터포털 데이터셋 설명 (웹)
3. 경쟁 지역 (부산, 강릉) 관광 통계
4. 관광 트렌드 리포트
5. 자체 분석 결과 (Markdown)
```

**Step 2: 질문**

```
질문:
"제주 관광의 강점과 약점을 다른 지역과 비교해서 분석해줘.
각 주장마다 어느 문서에서 나온 정보인지 출처를 명시해줘."
```

**NotebookLM 응답:**

```

1. 자연 경관 우수
```

## 4.16 팁과 베스트 프랙티스

### 4.16.1 1. 프롬프트 작성 팁

```
❌ "데이터 분석해줘"

```

**단계별로:**
```

1차: "데이터 개요 파악"
2차: "카테고리별 분석"
3차: "시각화"
4차: "인사이트 도출"

```

**출력 형식 지정:**
```
"결과를 다음 형식으로:

| 카테고리 | 평균 평점 | 개수 |
|---------|----------|------|
| ...     | ...      | ...  |

표 아래 3줄 요약 추가."
```

### 4.16.2 2. 데이터 업로드 팁

**파일 크기:**
```
ChatGPT: 최대 100MB
Claude: 최대 5개 파일 (각 5MB)
NotebookLM: 최대 50개 문서
```

**전처리:**
```
업로드 전:
- 불필요한 컬럼 제거

- 첫 행을 컬럼명으로
- 날짜 형식 통일 (YYYY-MM-DD)
```

### 4.16.3 3. 결과 검증

**항상 확인:**
```
✅ 계산 검증: 평균, 합계 다시 계산
```
