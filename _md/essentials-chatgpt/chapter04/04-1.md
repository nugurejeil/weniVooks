## 4.1 왜 제주 관광 데이터인가?

### 4.1.1 실습에 적합한 이유

**1. 친숙함**
- 대부분의 사람들이 제주도 방문 경험 보유
- 도메인 지식이 있어 데이터 이해가 쉬움
- 결과 해석이 직관적

**2. 데이터 풍부성**
- 공공데이터포털에서 무료로 제공
- 다양한 종류의 데이터 (관광지, 숙박, 음식점 등)
- 정기적으로 업데이트

**3. 실용성**
- 실제 비즈니스에 적용 가능한 인사이트
- 관광 산업은 데이터 분석이 중요한 분야
- 다른 지역/산업에도 응용 가능

## 4.2 제주 관광 데이터 개요

### 4.2.1 데이터 종류


## 4.3 데이터 탐색 프로세스

좋은 데이터 분석은 좋은 질문에서 시작합니다.

**비즈니스 질문 예시:**

```

정책 입안자 관점:
- 관광객 분산을 위해 어떤 지역을 개발해야 하나?
```

### 4.4.2 분석 질문과 방법

**질문 1: 가장 인기 있는 카테고리는?**

```
분석 방법:

```

**질문 2: 무료 vs 유료 관광지 비교**

```
분석 방법:
1. 무료/유료 관광지 개수
2. 각각의 평균 평점
3. 각각의 평균 리뷰 수

인사이트:
- 무료 관광지가 평점이 더 높다면?
  → 가성비 중시하는 관광객 많음
- 유료 관광지의 리뷰가 더 많다면?
  → 기대감이 높아 리뷰 작성 동기 높음
```

**질문 3: 지역별 관광지 분포**

```
분석 방법:
1. 동부/중부/서부별 관광지 수
2. 지역별 평균 평점
3. 지역별 카테고리 특성

인사이트:
- 특정 지역에 편중되어 있다면?
  → 관광객 분산 필요
- 지역별 특색이 있다면?
  → 테마별 관광 코스 기획 가능
```

## 4.5 데이터 품질 확인

### 4.5.1 체크 항목

**1. 완전성 (Completeness)**
```
□ 모든 필수 컬럼이 있는가?
□ 결측치 비율이 높지 않은가? (10% 이하 권장)
```

**2. 정확성 (Accuracy)**
```
□ 위도/경도가 제주도 범위 내인가?
  (위도: 33.1~33.6, 경도: 126.1~127.0)
□ 평점이 0~5 범위 내인가?
□ 입장료가 음수가 아닌가?
```

**3. 일관성 (Consistency)**
```
□ 카테고리 이름이 통일되어 있는가?
  예: "해변" vs "해수욕장" vs "비치"
□ 지역 구분이 명확한가?
□ 날짜 형식이 통일되어 있는가?
```

**4. 최신성 (Timeliness)**
```
□ 데이터 수집 시점은?
□ 최근 개장/폐쇄된 관광지가 반영되었는가?
```

