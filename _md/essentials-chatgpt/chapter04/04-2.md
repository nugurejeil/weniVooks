## 4.1 공공데이터포털 (data.go.kr)

### 4.1.1 공공데이터포털이란?

대한민국 정부가 운영하는 **국가 공공데이터 통합 플랫폼**입니다.

**특징:**
```
✅ 무료: 대부분의 데이터 무료 제공
✅ 다양성: 80,000+ 개의 데이터셋
✅ 품질: 정부 기관에서 직접 제공
✅ 법적 근거: 공공데이터법에 기반
✅ 상업적 이용: 대부분 가능 (라이선스 확인 필수)
```

**제공 형식:**
- 파일 데이터 (CSV, JSON, XML, Excel)
- Open API (RESTful API)
- 표준 데이터셋
- 시각화 자료

## 4.2 공공데이터포털 시작하기

### 4.2.1 1단계: 회원가입 및 로그인

**회원가입 절차:**

```
1. data.go.kr 접속
2. 우측 상단 "회원가입" 클릭
3. 회원 유형 선택:
   - 개인회원: 일반 사용자
   - 기업회원: 사업자등록증 보유
   - 공공회원: 공공기관
4. 본인인증 (휴대폰/아이핀)
5. 정보 입력 및 약관 동의
6. 가입 완료
```

**로그인:**
```
이메일 + 비밀번호
또는
간편 로그인 (카카오, 네이버 등)
```

### 4.2.2 2단계: 데이터 검색

**검색 방법:**

**1. 키워드 검색**
```
검색어: "제주 관광"
→ 관련 데이터셋 목록 표시

팁:
- 구체적인 키워드 사용: "제주 숙박" > "관광"
- 동의어 시도: "여행", "관광지", "명소"
- 지역명 포함: "제주도", "서귀포"
```

**2. 카테고리 탐색**
```
메인 카테고리:
├─ 공공행정
├─ 사회복지
├─ 교통물류
├─ 문화관광 ← 여기서 시작
├─ 환경기상
├─ 농축수산
└─ 기타

하위 카테고리:
문화관광 → 관광 → 관광지 정보
```

**3. 인기 데이터셋**
```
홈페이지 메인 > "인기 데이터"
- 다운로드 수 많은 데이터
- 활용도 높은 데이터
- 최근 업데이트된 데이터
```

### 4.2.3 3단계: 데이터 상세 정보 확인

**데이터셋 페이지 구성:**

```
[1] 기본 정보
- 제목
- 제공기관
- 업데이트 주기
- 마지막 업데이트 날짜

[2] 상세 설명
- 데이터 내용
- 포함 컬럼 (필드)
- 데이터 수집 방법
- 활용 예시

[3] 라이선스
- 이용 조건
- 상업적 이용 가능 여부
- 출처 표기 의무

[4] 제공 형식
- 파일 다운로드 (CSV, JSON, XML)
- Open API
- 미리보기

[5] 활용 사례
- 다른 사용자의 활용 사례
- 참고 자료
```

**확인해야 할 핵심 항목:**

```
□ 데이터 형식: CSV? JSON? API?
□ 데이터 양: 몇 건? (행 수)
□ 컬럼 정보: 어떤 필드가 있는가?
□ 업데이트 주기: 실시간? 월별? 연별?
□ 최신성: 마지막 업데이트가 언제?
□ 라이선스: 상업적 이용 가능한가?
□ API 키 필요 여부
```

### 4.2.4 4단계: 데이터 활용 신청

**파일 데이터 다운로드:**

```
1. 원하는 데이터셋 페이지 이동
2. "파일 데이터" 탭 클릭
3. 형식 선택 (CSV 권장)
4. "다운로드" 버튼 클릭
5. 로그인 필요 시 로그인
6. 다운로드 완료

→ 즉시 사용 가능!
```

**Open API 활용신청:**

```
1. "Open API" 탭 클릭
2. "활용신청" 버튼
3. 활용 목적 입력
   - 예: "개인 학습용 데이터 분석"
   - 예: "관광 추천 서비스 개발"
4. 활용 기간 선택
5. 신청 제출
6. 승인 대기 (대부분 즉시 승인)
7. 승인 후 "인증키" 발급받기
8. 인증키 복사 및 저장 (재발급 번거로움)
```

**API 키 관리:**

```
마이페이지 > 오픈API > 인증키 관리
- 발급받은 모든 키 확인
- 사용량 모니터링
- 키 재발급/삭제
```

## 4.4 데이터 형식 이해하기

### 4.4.1 CSV (Comma-Separated Values)

**특징:**
```
✅ 가장 보편적

❌ 복잡한 계층 구조 표현 어려움
```

**예시:**

**사용 도구:**

**특징:**
```
✅ 계층 구조 표현 가능
```

## 4.5 API 사용하기

### 4.5.1 API란?

**개념:**
```
API (Application Programming Interface)
= 프로그램이 데이터를 가져오는 방법

파일 다운로드 vs API:
- 파일: 한 번 다운로드, 고정된 데이터
- API: 실시간으로 최신 데이터 가져오기
```

**언제 API를 사용하나?**
```
✅ 데이터가 자주 업데이트될 때
✅ 필요한 일부만 가져오고 싶을 때
✅ 자동화하고 싶을 때
✅ 웹/앱 서비스를 만들 때

❌ 일회성 분석 → 파일 다운로드가 간편
```

### 4.5.2 API 요청 방법

**1. 웹 브라우저에서 테스트**

공공데이터포털 API 예시:

```
https://apis.data.go.kr/B551011/KorService/areaBasedList
?serviceKey=발급받은인증키
&numOfRows=10
&pageNo=1
&MobileOS=ETC
&MobileApp=AppTest
&areaCode=39
&_type=json
```

**URL 구조:**
```
https://apis.data.go.kr/B551011/KorService/areaBasedList
└─ 기본 주소 (엔드포인트)

?serviceKey=발급받은인증키
└─ 인증키 (필수)

&numOfRows=10
└─ 가져올 데이터 개수

&areaCode=39
└─ 지역 코드 (39=제주)

&_type=json
└─ 응답 형식 (json 또는 xml)
```

**브라우저에 붙여넣기:**
```
1. 위 URL에서 "발급받은인증키" 부분을 실제 키로 교체
2. 브라우저 주소창에 붙여넣기
3. Enter
4. JSON 데이터 응답 확인
```

**2. ChatGPT에게 요청하기**

ChatGPT는 직접 API 호출을 못하지만, 코드를 생성해줄 수 있습니다.

**프롬프트:**
```
다음 공공데이터 API를 Python으로 호출하는 코드를 작성해줘:

API 주소: https://apis.data.go.kr/B551011/KorService/areaBasedList
인증키: [내 인증키]
파라미터:
- numOfRows: 100
- areaCode: 39 (제주)
- contentTypeId: 12 (관광지)

요청을 보내고, JSON 응답을 파싱해서 pandas DataFrame으로 변환한 후
CSV 파일로 저장하는 전체 코드를 작성해줘.
```

**3. Python 코드 예시**

```python
import requests
import pandas as pd

# API 정보
api_url = "https://apis.data.go.kr/B551011/KorService/areaBasedList"
service_key = "발급받은인증키"  # 본인의 키로 교체

# 파라미터
params = {
    'serviceKey': service_key,
    'numOfRows': 100,
    'pageNo': 1,
    'MobileOS': 'ETC',
    'MobileApp': 'AppTest',
    'areaCode': 39,  # 제주
    'contentTypeId': 12,  # 관광지
    '_type': 'json'
}

# API 요청
response = requests.get(api_url, params=params)
data = response.json()

# 데이터 추출
items = data['response']['body']['items']['item']

# DataFrame 변환
df = pd.DataFrame(items)

# CSV 저장
df.to_csv('제주관광지.csv', index=False, encoding='utf-8-sig')

print(f"총 {len(df)}개의 관광지 정보를 저장했습니다.")
print(df.head())
```

### 4.5.3 API 응답 데이터 이해하기

**일반적인 JSON 응답 구조:**

```json
{
  "response": {
    "header": {
      "resultCode": "0000",
      "resultMsg": "OK"
    },
    "body": {
      "items": {
        "item": [
          {
            "title": "성산일출봉",
            "addr1": "제주특별자치도 서귀포시 성산읍",
            "mapx": "126.9425",
            "mapy": "33.4586"
          },
          {
            "title": "만장굴",
            ...
          }
        ]
      },
      "numOfRows": 10,
      "pageNo": 1,
      "totalCount": 347
    }
  }
}
```

**핵심 부분:**
```
response.header.resultCode
→ "0000" = 성공, 그 외 = 오류

response.body.items.item
→ 실제 데이터 배열

response.body.totalCount
→ 전체 데이터 개수
```

**페이지네이션:**

API는 한 번에 모든 데이터를 주지 않습니다.

```python
# 전체 데이터 가져오기 (여러 페이지)
all_data = []
page = 1

while True:
    params['pageNo'] = page
    response = requests.get(api_url, params=params)
    data = response.json()

    items = data['response']['body']['items']['item']

    if not items:  # 더 이상 데이터 없음
        break

    all_data.extend(items)
    page += 1

    print(f"페이지 {page-1} 완료: {len(items)}개")

df = pd.DataFrame(all_data)
```

## 4.6 여러 데이터 소스 결합하기

### 4.6.1 왜 데이터를 결합하나?

단일 데이터셋으로는 부족할 때가 많습니다.

**예시:**
```
질문: "제주에서 숙박비가 저렴하면서 관광지가 많은 지역은?"

필요한 데이터:
1. 숙박업소 데이터 (가격, 위치)
2. 관광지 데이터 (위치)
→ 두 데이터를 지역별로 결합 필요
```

### 4.6.2 결합 방법 1: 수동 결합 (Excel)

**간단한 경우:**

```
데이터 A: 제주 지역별 관광지 수
지역,관광지수
```

### 4.6.4 결합 방법 3: Python pandas

```python
import pandas as pd

# 데이터 로드
df_tourism = pd.read_csv('관광지.csv')
df_hotel = pd.read_csv('숙박.csv')

# 결합 (지역 기준)
df_merged = pd.merge(
    df_tourism,
    df_hotel,
    on='지역',  # 공통 컬럼
    how='inner'  # inner, left, right, outer
)

# 결과 저장
df_merged.to_csv('결합데이터.csv', index=False)
```

**결합 타입:**
```
inner: 양쪽 모두에 있는 데이터만
left: 왼쪽 데이터는 모두, 오른쪽은 매칭되는 것만
right: 오른쪽 데이터는 모두, 왼쪽은 매칭되는 것만
outer: 양쪽 모두 포함 (빈 곳은 NaN)
```

### 4.6.5 실전 예제: 제주 관광 종합 데이터셋 만들기

**목표:**
```
제주 관광 분석을 위한 종합 데이터셋
= 관광지 + 숙박 + 음식점 + 방문객 통계
```

**Step 1: 각 데이터 수집**

```
1. 제주 관광지 정보 (data.go.kr)
2. 제주 숙박업소 정보 (data.go.kr)
3. 제주 음식점 정보 (data.go.kr)
4. 제주 방문객 통계 (제주관광공사)
```

**Step 2: 데이터 정제**

```python
# 1. 관광지 데이터
df_tour = pd.read_csv('관광지.csv')
df_tour = df_tour[['이름', '주소', '위도', '경도', '카테고리', '평점']]
df_tour['지역'] = df_tour['주소'].str.split().str[0]  # 주소에서 지역 추출

# 2. 숙박 데이터
df_hotel = pd.read_csv('숙박.csv')
df_hotel = df_hotel[['이름', '주소', '숙박유형', '가격대']]
df_hotel['지역'] = df_hotel['주소'].str.split().str[0]

# 3. 음식점 데이터
df_food = pd.read_csv('음식점.csv')
df_food = df_food[['이름', '주소', '카테고리', '평점']]
df_food['지역'] = df_food['주소'].str.split().str[0]
```

**Step 3: 지역별 집계**

```python
# 지역별 관광지 수
tour_count = df_tour.groupby('지역').size().reset_index(name='관광지수')

# 지역별 평균 숙박비
hotel_avg = df_hotel.groupby('지역')['가격대'].mean().reset_index(name='평균숙박비')

# 지역별 음식점 수
food_count = df_food.groupby('지역').size().reset_index(name='음식점수')
```

**Step 4: 결합**

```python
# 순차적으로 결합
df_final = tour_count
df_final = pd.merge(df_final, hotel_avg, on='지역', how='left')
df_final = pd.merge(df_final, food_count, on='지역', how='left')

# 방문객 데이터 추가 (시계열 데이터는 별도 처리)
df_visitor = pd.read_csv('방문객.csv')
# ...추가 로직

# 최종 저장
df_final.to_csv('제주관광_종합데이터.csv', index=False)
```
