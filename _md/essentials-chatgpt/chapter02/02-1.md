## 2.1 인공지능이란?

인공지능(Artificial Intelligence, AI)은 인간의 지능을 모방하여 학습, 추론, 문제 해결 등을 수행하는 컴퓨터 시스템을 말합니다.

### 2.1.1 AI의 역사

**1950년대: AI의 탄생**
- 1950년: 앨런 튜링의 "튜링 테스트" 제안
- 1956년: 다트머스 회의에서 "인공지능"이라는 용어 최초 사용
- 목표: 인간처럼 생각하는 기계 만들기

**1960-1970년대: 첫 번째 AI 겨울**
- 초기 기대에 비해 실망스러운 결과
- 컴퓨팅 파워 부족
- 연구 자금 축소

**1980-1990년대: 전문가 시스템의 시대**
- 규칙 기반 AI 시스템 발전
- 체스 AI가 인간 챔피언 격파 (1997년 딥블루)
- 실용적 AI 응용 시작

**2000년대: 빅데이터와 머신러닝**
- 인터넷 보급으로 데이터 폭발적 증가
- 컴퓨팅 파워 향상
- 머신러닝 알고리즘 발전

**2010년대: 딥러닝 혁명**
- 2012년: AlexNet, 이미지 인식 경진대회 우승
- 2016년: 알파고, 이세돌 9단 격파
- GPU를 활용한 대규모 학습 가능

**2020년대: 생성형 AI 시대**
- 2022년: ChatGPT 공개, AI 대중화
- 2023-2024년: 멀티모달 AI 등장
- **현재: 모든 산업에 AI 통합**

### 2.1.2 AI의 분류

AI는 능력에 따라 다음과 같이 분류됩니다:

```
┌─────────────────────────────────────┐
│      약인공지능 (Narrow AI)          │
│  특정 작업만 수행 (현재 우리가 사용)  │
│  예: ChatGPT, 이미지 인식, 번역      │
└─────────────────────────────────────┘
         ↓ (아직 미달성)
┌─────────────────────────────────────┐
│      강인공지능 (General AI)         │
│  인간 수준의 범용 지능               │
│  모든 분야에서 인간처럼 사고          │
└─────────────────────────────────────┘
         ↓ (먼 미래)
┌─────────────────────────────────────┐
│      초인공지능 (Super AI)           │
│  인간을 넘어서는 지능                │
│  모든 분야에서 인간을 초월            │
└─────────────────────────────────────┘
```

**현재 우리가 사용하는 모든 AI는 "약인공지능"입니다.**

## 2.2 머신러닝이란?

머신러닝(Machine Learning, ML)은 명시적으로 프로그래밍하지 않고도 데이터로부터 학습하여 작업을 수행하는 AI의 한 분야입니다.

### 2.2.1 전통적 프로그래밍 vs 머신러닝

**전통적 프로그래밍:**
```
입력 데이터 + 규칙(코드) → 결과

예시: 스팸 필터
IF 메일에 "무료"가 포함되고
   발신자가 수상하면
   → 스팸으로 분류
```

**머신러닝:**
```
입력 데이터 + 정답 → 학습 → 규칙(모델)

예시: 스팸 필터
수천 개의 메일(스팸/정상)을 학습
→ AI가 스스로 패턴 발견
→ 새로운 메일 자동 분류
```

### 2.2.2 머신러닝의 종류

**1. 지도 학습 (Supervised Learning)**

정답이 있는 데이터로 학습합니다.

```
훈련 데이터:
- 이미지 + 라벨(고양이/강아지)
- 메일 + 라벨(스팸/정상)
- 집 정보 + 가격

→ 학습 → 새로운 데이터 예측
```

**실생활 예시:**
- 스팸 메일 필터링
- 질병 진단 (X-ray 이미지 → 병명)
- 부동산 가격 예측
- 신용카드 사기 탐지

**2. 비지도 학습 (Unsupervised Learning)**

정답 없이 데이터의 패턴을 스스로 찾습니다.

```
훈련 데이터:
- 고객 구매 데이터 (라벨 없음)

→ 학습 → 고객 그룹 자동 분류
         (VIP 고객, 일반 고객 등)
```

**실생활 예시:**
- 고객 세분화 (Customer Segmentation)
- 이상 탐지 (Anomaly Detection)
- 추천 시스템 (넷플릭스, 유튜브)

**3. 강화 학습 (Reinforcement Learning)**

보상과 처벌을 통해 학습합니다.

```
행동 → 환경 반응 → 보상/처벌 → 학습

예: 게임 AI
좋은 행동 → 점수 획득 (보상)
나쁜 행동 → 게임 오버 (처벌)
→ 최적의 전략 학습
```

**실생활 예시:**
- 게임 AI (알파고, 스타크래프트 AI)
- 자율주행 자동차
- 로봇 제어
- 주식 거래 AI

### 2.2.3 머신러닝 학습 과정

**1단계: 데이터 수집**
```
예: 고양이 사진 1만 장 수집
```

**2단계: 데이터 전처리**
```
- 이미지 크기 통일
- 노이즈 제거
- 라벨링 (고양이/강아지 표시)
```

**3단계: 모델 선택**
```
- 이미지 분류에 적합한 모델 선택
- 예: CNN (Convolutional Neural Network)
```

**4단계: 학습 (Training)**
```
데이터를 반복해서 학습
→ 정확도 점점 향상
```

**5단계: 평가 (Evaluation)**
```
학습하지 않은 새로운 데이터로 테스트
→ 정확도 측정
```

**6단계: 배포 (Deployment)**
```
실제 서비스에 적용
```

## 2.3 딥러닝이란?

딥러닝(Deep Learning)은 인간의 뇌 구조를 모방한 인공신경망을 여러 층으로 쌓아 학습하는 머신러닝의 한 분야입니다.

### 2.3.1 AI, ML, DL의 관계

```
┌─────────────────────────────────────┐
│       인공지능 (AI)                  │
│   ┌─────────────────────────────┐   │
│   │   머신러닝 (ML)               │   │
│   │  ┌─────────────────────┐    │   │
│   │  │  딥러닝 (DL)         │    │   │
│   │  │  - CNN              │    │   │
│   │  │  - RNN              │    │   │
│   │  │  - Transformer      │    │   │
│   │  └─────────────────────┘    │   │
│   └─────────────────────────────┘   │
└─────────────────────────────────────┘
```

**핵심 개념:**
- AI가 가장 넓은 개념
- ML은 AI의 부분집합
- DL은 ML의 부분집합

### 2.3.2 인공신경망의 원리

**뉴런(Neuron)의 작동:**

```
입력1 ──┐
입력2 ──┼─→ [뉴런] ─→ 출력
입력3 ──┘
       ↑
    가중치(weight)
```

**예시: 스팸 메일 판단**
```
"무료" 포함 (0.7) ──┐
수상한 발신자 (0.9)──┼─→ [뉴런] ─→ 0.85 → 스팸!
링크 많음 (0.6) ────┘

* 숫자는 가중치(중요도)
```

**다층 신경망 (Deep Neural Network):**

```
입력층    은닉층1   은닉층2   은닉층3    출력층
  ●        ●        ●         ●         ●
  ●    ╱ ╲   ●    ╱ ╲  ●    ╱ ╲  ●    ╱  ● (고양이 90%)
  ●   ╱   ╲  ●   ╱   ╲ ●   ╱   ╲ ●  ╱   ● (강아지 10%)
  ●  ╱     ╲ ●  ╱     ╲●  ╱     ╲● ╱
  ● ╱       ╲● ╱       ●╱       ●╱

→ 층이 깊어질수록 복잡한 패턴 학습 가능
```

### 2.3.3 딥러닝의 장점과 단점

**장점:**
- **자동 특징 추출**: 사람이 특징을 지정하지 않아도 AI가 스스로 찾음
- **높은 정확도**: 대량의 데이터로 학습하면 사람보다 정확
- **확장성**: 더 많은 데이터와 컴퓨팅으로 성능 향상

**단점:**
- **대량의 데이터 필요**: 수천~수백만 개의 학습 데이터 필요
- **높은 컴퓨팅 비용**: 강력한 GPU 필요
- **블랙박스**: 왜 그런 결정을 내렸는지 설명하기 어려움

### 2.3.4 주요 딥러닝 모델

**1. CNN (Convolutional Neural Network)**
- **용도**: 이미지 인식, 영상 처리
- **특징**: 이미지의 패턴을 단계적으로 학습
- **예시**: 얼굴 인식, 자율주행, 의료 영상 분석

**2. RNN (Recurrent Neural Network)**
- **용도**: 시계열 데이터, 순차 데이터
- **특징**: 이전 정보를 기억하며 학습
- **예시**: 주가 예측, 음성 인식, 기계 번역

**3. Transformer**
- **용도**: 자연어 처리 (NLP)
- **특징**: 문맥을 병렬로 처리하여 빠르고 정확
- **예시**: ChatGPT, Claude, 번역기
- **혁신**: 생성형 AI의 핵심 기술

## 2.4 생성형 AI로 가는 길

### 2.4.1 Transformer의 등장 (2017년)

Google이 발표한 논문 "Attention is All You Need"에서 Transformer 구조 제안

**핵심 메커니즘: Attention (주의 집중)**

```
문장: "나는 강에 갔다. 그곳은 아름다웠다."

기존 RNN:
"그곳" → 바로 이전 단어만 참고

Transformer Attention:
"그곳" → 문장 전체를 보고 "강"을 가리킴을 파악
```

**혁신적인 점:**
- 문장 전체를 동시에 처리 (병렬 처리)
- 문맥을 더 잘 이해
- 학습 속도가 빠름

### 2.4.2 GPT의 등장

**GPT-1 (2018년)**
- Transformer 기반 언어 모델
- 책 7,000권 분량 학습
- 문장 생성 능력 보임

**GPT-2 (2019년)**
- 40GB 텍스트 학습
- 너무 그럴싸한 가짜 뉴스 생성 우려로 공개 보류

**GPT-3 (2020년)**
- 175억 개 파라미터
- 웹 전체의 상당 부분 학습
- 놀라운 언어 이해와 생성 능력

**ChatGPT (2022년 11월)**
- GPT-3.5 기반
- 대화형 인터페이스
- **5일 만에 100만 사용자 돌파**
- **AI 대중화의 시작**

## 2.5 실무에서 알아야 할 핵심 개념

### 2.5.1 1. 파라미터 (Parameters)

모델의 "뇌세포" 개수라고 생각하면 됩니다.

```
GPT-3: 1,750억 개
GPT-4: 약 1조 개 (추정)

파라미터가 많을수록:
- 더 복잡한 패턴 학습 가능
- 더 많은 지식 저장 가능
- 더 많은 컴퓨팅 파워 필요
```

### 2.5.2 2. 토큰 (Tokens)

AI가 텍스트를 처리하는 기본 단위입니다.

```
"안녕하세요" → 약 3-4 토큰
"Hello" → 1 토큰

ChatGPT Plus:
- 한 번에 약 8,000 토큰 처리 가능
- 입력 + 출력 포함

Claude:
- 한 번에 약 200,000 토큰 처리 가능
- 긴 문서 분석에 유리
```

### 2.5.3 3. 학습 (Training) vs 추론 (Inference)

**학습 (Training):**
```
- OpenAI, Anthropic 같은 회사가 수행
- 수백만 달러의 비용
- 수천 개의 GPU로 몇 개월
- 우리는 할 필요 없음
```

**추론 (Inference):**
```
- 우리가 ChatGPT를 사용하는 것
- 이미 학습된 모델로 답변 생성
- 비용: 월 2만원
```

### 2.5.4 4. Hallucination (환각)

AI가 그럴싸하지만 틀린 정보를 생성하는 현상

**원인:**
- 학습 데이터에 없는 정보 요청
- 불확실한 상황에서도 답변 생성
- 사실과 추론을 구분하지 못함

**대응 방법:**
- 중요한 정보는 반드시 검증
- 출처를 함께 요청
- 최신 정보는 검색 도구 활용

### 2.5.5 5. Fine-tuning vs Prompt Engineering

**Fine-tuning (미세 조정):**
```
- 모델 자체를 재학습
- 비용: 수천만~수억 원
- 기업용 맞춤 AI 제작 시

예: 법률 전문 AI, 의료 전문 AI
```

**Prompt Engineering (프롬프트 엔지니어링):**
```
- 질문 방식만 개선
- 비용: 무료
- 일반 사용자가 활용

예: "마케팅 전문가처럼 답변해줘"
```

**대부분의 경우 Prompt Engineering으로 충분합니다!**

## 2.6 실무 활용 관점에서의 정리

### 2.6.1 일반 사용자가 알아야 할 것

**알아야 할 것:**
- ✅ AI의 기본 개념 (이 페이지 내용)
- ✅ 생성형 AI의 강점과 한계
- ✅ Hallucination 현상과 대응
- ✅ 프롬프트 작성 방법

**몰라도 되는 것:**
- ❌ 신경망 수식
- ❌ 역전파 알고리즘
- ❌ 최적화 알고리즘
- ❌ 모델 학습 방법

### 2.6.2 핵심 요약

```
1. AI는 데이터로부터 학습하는 시스템
2. 머신러닝은 AI의 한 분야
3. 딥러닝은 머신러닝의 한 분야
4. Transformer는 생성형 AI의 핵심 기술
5. 우리는 "사용자"로서 잘 활용하면 됨
```

**중요한 것은 기술이 아니라 활용 능력입니다!**

## 2.7 마무리

AI와 머신러닝의 기본 개념을 이해했다면, 다음 섹션에서는 생성형 AI가 어떻게 발전해왔는지 구체적으로 살펴보겠습니다.

**기억해야 할 것:**
- AI는 더 이상 전문가만의 영역이 아닙니다
- ChatGPT, Claude 같은 생성형 AI는 누구나 사용 가능
- 기술 원리보다 **어떻게 활용할지**가 더 중요합니다
