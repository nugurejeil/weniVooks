## 2.1 생성형 AI의 분류

생성형 AI는 생성하는 콘텐츠 유형에 따라 분류할 수 있습니다.

```
생성형 AI
├── 텍스트 생성 (LLM: Large Language Model)
│   ├── ChatGPT, Claude, Gemini
│   └── 글쓰기, 번역, 코드, 대화 등
├── 이미지 생성
│   ├── Midjourney, DALL-E, Stable Diffusion
│   └── 일러스트, 사진, 디자인 등
├── 영상 생성
│   ├── Sora, Runway, Pika
│   └── 단편 영상, 애니메이션 등
├── 음악 생성
│   ├── Suno, Udio
│   └── 작곡, 작사, 보컬 등
├── 음성 생성 (TTS)
│   ├── ElevenLabs, PlayHT
│   └── 나레이션, 성우 등
└── 코드 생성
    ├── GitHub Copilot, Cursor, Claude Code
    └── 프로그래밍 자동화
```

## 2.2 1. 텍스트 생성 AI (LLM)

### 2.2.1 주요 모델 비교

| 모델 | 개발사 | 강점 | 약점 | 가격 |
|------|--------|------|------|------|
| **ChatGPT** | OpenAI | 대중적, 사용 쉬움 | 최신 정보 부족 | 월 $20 |
| **Claude** | Anthropic | 긴 문서, 정확성 | 이미지 생성 없음 | 월 $20 |
| **Gemini** | Google | 검색 연동, 무료 | 한국어 약함 | 무료/월 $20 |
| **Perplexity** | Perplexity AI | 실시간 검색 | 창의성 부족 | 무료/월 $20 |

### 2.2.2 활용 분야

**1. 글쓰기**

```
프롬프트: "스타트업 제품 소개 블로그 글을 800자로 써줘"

AI가 할 수 있는 것:
- 초안 작성 (50-70% 완성도)
- 구조화 및 개요 작성
- 톤앤매너 조정
- 다양한 버전 생성

사람이 해야 할 것:
- 사실 확인
- 브랜드 정체성 반영
- 최종 편집 및 다듬기
```

**예시:**
- 블로그 글, 보고서, 이메일
- 소셜 미디어 게시물
- 광고 카피, 제품 설명

**2. 번역**

```
기존 번역기 (Google 번역):
직역 위주, 문맥 이해 부족

생성형 AI 번역:
문맥 이해, 자연스러운 번역, 톤 조정 가능

프롬프트:
"다음 영어 문장을 비즈니스 이메일 톤으로 번역해줘"
→ 격식 있고 자연스러운 번역
```

**장점:**
- 전문 용어 정확도 높음
- 문화적 뉘앙스 반영
- 여러 언어 동시 번역

**3. 요약**

```
입력: 20페이지 보고서
출력: 핵심 내용 1페이지 요약

활용:
- 회의록 요약
- 논문/기사 요약
- 계약서 핵심 조항 추출
```

**Claude의 장문 요약:**
- 200,000 토큰 (약 150,000 단어)
- 책 여러 권을 한 번에 요약 가능

**4. 코드 생성**

```
프롬프트: "Python으로 CSV 파일을 읽어서
          월별 매출 그래프를 그리는 코드 짜줘"

AI 생성 코드:
import pandas as pd
import matplotlib.pyplot as plt

df = pd.read_csv('sales.csv')
monthly_sales = df.groupby('month')['sales'].sum()
monthly_sales.plot(kind='bar')
plt.show()
```

**활용:**
- 간단한 스크립트 작성
- 알고리즘 구현
- 버그 수정
- 테스트 코드 생성

**5. 데이터 분석**

```
CSV 업로드 후:

"이 데이터에서 매출이 가장 높은 제품 3개를 찾고,
 월별 트렌드를 분석해줘"

AI가 수행:
1. 데이터 정제
2. 통계 분석
3. 시각화 (그래프)
4. 인사이트 도출
```

**ChatGPT Advanced Data Analysis:**
- Python 코드 자동 실행
- 그래프 자동 생성
- 복잡한 분석 가능

### 2.2.3 실전 팁

**1. 명확한 지시**
```
나쁜 예: "블로그 글 써줘"
좋은 예: "스타트업 CEO를 대상으로
         AI 도입 방법을 설명하는
         800자 블로그 글을 친근한 톤으로 써줘"
```

**2. 단계별 요청**
```
1단계: "마케팅 이메일 제목 5개 제안해줘"
2단계: "2번 제목으로 본문 작성해줘"
3단계: "좀 더 급박한 느낌으로 수정해줘"
```

**3. 예시 제공**
```
"다음과 같은 스타일로 써줘:
 [이전에 잘 쓴 글 붙여넣기]"
```

## 2.3 2. 이미지 생성 AI

### 2.3.1 주요 모델 비교

| 모델 | 강점 | 약점 | 가격 | 적합한 용도 |
|------|------|------|------|-------------|
| **Midjourney** | 예술적 품질 | 텍스트 렌더링 약함 | 월 $10~ | 콘셉트 아트, 일러스트 |
| **DALL-E 3** | 프롬프트 이해력 | 스타일 제한적 | ChatGPT Plus 포함 | 사실적 이미지 |
| **Stable Diffusion** | 커스터마이징 | 높은 학습 곡선 | 무료 | 전문가용 |
| **Firefly** | Adobe 통합 | 창의성 부족 | 월 $5~ | 상업용 안전 |

### 2.3.2 프롬프트 작성법

**기본 구조:**
```
[주제] + [스타일] + [디테일] + [기술적 요소]

예시:
"a cat (주제)
in cyberpunk style (스타일)
wearing neon glasses (디테일)
8k, highly detailed, digital art (기술적 요소)"
```

**좋은 프롬프트의 요소:**

1. **명확한 주제**
```
나쁜 예: "pretty landscape"
좋은 예: "mountain landscape at sunset with a lake"
```

2. **구체적인 스타일**
```
- 사실적: photorealistic, realistic, 8k
- 일러스트: illustration, digital art, cartoon
- 예술적: oil painting, watercolor, impressionist
```

3. **분위기와 조명**
```
- dramatic lighting
- soft ambient light
- golden hour
- moody, dark atmosphere
```

4. **카메라 앵글**
```
- close-up portrait
- wide angle shot
- bird's eye view
- cinematic shot
```

### 2.3.3 활용 사례

**1. 마케팅 소재**

```
프롬프트:
"professional business team meeting,
modern office, natural lighting,
diverse people, photorealistic"

용도:
- 웹사이트 배너
- 소셜 미디어 포스트
- 프레젠테이션 배경
```

**2. 제품 콘셉트**

```
프롬프트:
"futuristic smartphone design,
transparent screen, holographic display,
product photography, white background"

용도:
- 제품 개발 초기 콘셉트
- 투자 유치 자료
- 고객 설문용 목업
```

**3. 콘텐츠 제작**

```
용도:
- 블로그 썸네일
- YouTube 썸네일
- 책 표지
- 캐릭터 디자인
```

### 2.3.4 저작권 이슈

**주의사항:**
- 유명인 얼굴 생성 금지
- 상표/로고 무단 사용 금지
- 상업적 사용 시 라이선스 확인

**안전한 사용:**
- Adobe Firefly: 상업적 사용 보장
- Midjourney: 유료 플랜은 상업적 사용 가능
- DALL-E: OpenAI 정책 준수

## 2.4 3. 영상 생성 AI

### 2.4.1 주요 도구

**1. Sora (OpenAI)**
- 텍스트로 1분 영상 생성
- 현재 대기자 명단
- 품질: 영화 수준

**2. Runway**
- 기존 영상 편집
- 배경 제거, 객체 추가
- Gen-2: 텍스트로 영상 생성

**3. Pika**
- 간단한 영상 생성
- 이미지를 영상으로 변환
- 무료 플랜 있음

### 2.4.2 현재 한계

```
잘하는 것:
- 짧은 클립 (5-10초)
- 풍경, 추상적 장면
- 카메라 워크

어려운 것:
- 긴 영상 (일관성 유지)
- 사람 움직임 (부자연스러움)
- 복잡한 장면 전환
```

### 2.4.3 실전 활용

**현재 가능한 용도:**
- 소셜 미디어 짧은 광고
- 제품 티저 영상
- 프레젠테이션 애니메이션
- 스톡 영상 대체

**아직 어려운 것:**
- 장편 영상
- 드라마/영화
- 정밀한 편집 필요한 작업

## 2.9 미래 전망

### 2.9.1 곧 상용화될 기술

**1. 멀티모달 통합 (1-2년)**
```
현재: 텍스트 → 이미지 → 영상 (따로따로)
미래: "광고 영상 만들어줘" →
      스크립트 + 이미지 + 영상 + 음악 한 번에
```

**2. 개인화 (2-3년)**
```
현재: 모두에게 같은 AI
미래: 나의 업무 스타일을 학습한 AI
      나만의 프롬프트 템플릿 자동 생성
```

**3. 실시간 협업 (3-5년)**
```
현재: AI에게 요청 → 결과 받음
미래: AI와 실시간으로 함께 작업
      "여기는 이렇게 바꿔봐" → 즉시 반영
```

### 2.9.2 준비해야 할 것

**1. 지금 배워야 할 스킬**
- 프롬프트 엔지니어링
- AI 결과물 평가 능력
- AI 윤리와 한계 이해

**2. 변하지 않는 가치**
- 창의성과 기획력
- 비판적 사고
- 인간관계와 소통

**3. 새로운 기회**
- AI 도구 컨설턴트
- 프롬프트 디자이너
- AI 품질 관리자

## 2.11 정리

### 2.11.1 핵심 메시지

**1. 올인원 도구는 없다**
- 각 AI는 특화 분야가 다름
- 조합해서 사용하는 것이 효과적

**2. 도구는 계속 진화한다**
- 매달 새로운 기능 추가
- 지속적인 학습 필요
- 트렌드 팔로우 중요

**3. AI는 보조 도구**
- 최종 판단은 사람
- 창의성의 시작점
- 생산성 향상 수단

**4. 지금 시작하는 것이 중요**
- 늦을수록 격차 증가
- 실전 경험이 가장 중요
