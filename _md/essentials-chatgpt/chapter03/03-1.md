## 3.1 프롬프트란?

프롬프트(Prompt)는 AI에게 주는 명령어나 질문입니다. 프롬프트를 어떻게 작성하느냐에 따라 결과물의 품질이 크게 달라집니다.

### 3.1.1 좋은 프롬프트의 4가지 요소

```
1. 역할 (Role): AI가 어떤 역할을 해야 하는지
2. 맥락 (Context): 배경 정보와 상황 설명
3. 작업 (Task): 구체적으로 무엇을 해야 하는지
4. 형식 (Format): 결과물의 형태와 스타일
```

**나쁜 프롬프트:**
```
"마케팅 글 써줘"
```

**좋은 프롬프트:**
```
역할: 너는 B2B SaaS 마케팅 전문가야.
맥락: 우리는 중소기업 대상 프로젝트 관리 툴을 출시했어.
작업: 제품 출시 이메일을 작성해줘.
형식: 제목(10자 이내) + 본문(200자) + CTA 버튼 문구
```

## 3.2 1. 마케팅 분야 프롬프트

### 3.2.1 1.1 블로그 글 작성

**템플릿:**
```
[역할]
너는 [산업/분야]의 전문 콘텐츠 마케터야.

[맥락]
- 타겟 독자: [구체적인 독자층]
- 목적: [인지도 향상/리드 생성/교육 등]
- 톤앤매너: [전문적/친근한/유머러스 등]

[작업]
다음 주제로 블로그 글을 써줘: [주제]

[형식]
- 제목: 클릭을 유도하는 제목
- 도입부: 독자의 관심을 끄는 질문이나 통계
- 본문: 3-5개 소제목으로 구조화
- 결론: 명확한 CTA (Call To Action)
- 길이: 약 [글자 수]
```

**실전 예시:**
```
너는 스타트업 마케팅 전문가야.

타겟 독자는 마케팅 예산이 적은 초기 스타트업 대표들이야.
목적은 우리 마케팅 툴의 무료 체험을 신청하게 하는 거야.
톤은 친근하면서도 전문적이어야 해.

"작은 예산으로 큰 효과를 내는 그로스 해킹 전략"이라는 주제로
블로그 글을 써줘.

형식:
- 제목: 클릭률 높은 제목
- 도입부: 스타트업의 마케팅 예산 고민 공감
- 본문: 5가지 실천 가능한 그로스 해킹 전략
- 결론: 우리 툴 무료 체험 CTA
- 길이: 1,500자
```

### 3.2.2 1.2 소셜 미디어 콘텐츠

**템플릿:**
```
[플랫폼]: [인스타그램/트위터/링크드인]
[목표]: [팔로워 증가/참여 유도/링크 클릭]
[제품/서비스]: [간단한 설명]

다음 내용으로 게시물을 만들어줘:
- 주제: [...]
- 포함할 해시태그 수: [...]
- 이모지 사용: [많이/적당히/안 함]
```

**실전 예시 - 인스타그램:**
```
플랫폼: 인스타그램
목표: 참여율 높이기 (좋아요, 댓글, 저장)
제품: 비건 단백질 쉐이크

주제: 운동 후 단백질 섭취 타이밍
해시태그: 5개
이모지: 적당히 사용
형식: 캐러셀 첫 페이지 텍스트 (5줄 이내)
```

**실전 예시 - 링크드인:**
```
플랫폼: 링크드인
목표: 소트 리더십 확립 및 웹사이트 트래픽
주제: AI가 마케팅 업무를 어떻게 바꾸고 있는지

톤: 전문적이지만 접근 가능한
길이: 150-200단어
구조:
1. Hook (눈길 끄는 첫 문장)
2. 개인적 경험이나 인사이트
3. 실용적인 조언 2-3가지
4. 질문으로 토론 유도
```

### 3.2.3 1.3 광고 카피

**템플릿:**
```
[제품/서비스]: [...]
[타겟 고객]: [...]
[핵심 메시지]: [...]
[플랫폼]: [Google Ads/Facebook/Instagram]

다음 형식으로 광고 카피를 작성해줘:
- 헤드라인: [글자 수 제한]
- 바디 카피: [글자 수 제한]
- CTA: [클릭 유도 문구]
```

**실전 예시:**
```
제품: 온라인 영어 회화 수업
타겟: 영어 회화에 자신 없는 직장인 (25-35세)
핵심 메시지: 원어민 강사와 1:1로 부담 없이 연습
플랫폼: Facebook 광고

형식:
- 헤드라인: 40자 이내, 고객 pain point 건드리기
- 바디: 100자, 솔루션과 혜택 제시
- CTA: "무료 체험 신청하기"

5가지 버전을 A/B 테스트용으로 만들어줘.
```

### 3.2.4 1.4 이메일 마케팅

**템플릿:**
```
[이메일 유형]: [환영/프로모션/재참여/뉴스레터]
[받는 사람]: [신규 가입자/기존 고객/휴면 고객]
[목표]: [...]

구조:
- 제목: [...]
- 프리헤더: [...]
- 본문: [...]
- CTA: [...]
```

**실전 예시 - 환영 이메일:**
```
이메일 유형: 환영 이메일 (시리즈 1/3)
받는 사람: 방금 무료 체험을 신청한 신규 사용자
목표: 첫 로그인 유도 및 핵심 기능 소개

제목: 개인화된 제목 (이름 포함), 궁금증 유발
프리헤더: 제목을 보완하는 한 문장
본문:
  - 따뜻한 환영 메시지
  - 첫 3일 동안 해야 할 3가지 (단계별 안내)
  - 각 단계별 명확한 버튼
CTA: "첫 프로젝트 만들기" (메인 버튼)

톤: 친근하고 도움이 되는
길이: 짧고 간결하게 (150단어 이내)
```

## 3.3 2. 개발 분야 프롬프트

### 3.3.1 2.1 코드 작성

**템플릿:**
```
[언어]: [Python/JavaScript/Java 등]
[목적]: [무엇을 하는 코드인지]

요구사항:
- [...]
- [...]

제약사항:
- [...]

추가 요청:
- 주석 포함 여부
- 에러 핸들링 여부
- 테스트 코드 필요 여부
```

**실전 예시:**
```
언어: Python
목적: CSV 파일을 읽어서 데이터 정제 및 분석

요구사항:
- CSV 파일에서 결측치 제거
- 특정 컬럼의 이상치 탐지 (IQR 방식)
- 월별 집계 후 그래프 출력
- 결과를 새로운 CSV로 저장

제약사항:
- pandas와 matplotlib 사용
- 함수형으로 작성 (재사용 가능하게)

추가 요청:
- 각 함수에 docstring 포함
- try-except로 에러 핸들링
- 사용 예시 코드도 함께 작성
```

### 3.3.2 2.2 버그 수정

**템플릿:**
```
[문제 설명]: [어떤 에러가 발생하는지]
[에러 메시지]: [실제 에러 메시지]

코드:
```[언어]
[코드 붙여넣기]
```

기대 동작: [...]
실제 동작: [...]
```

**실전 예시:**
```
문제: JavaScript에서 비동기 함수 호출 시 undefined 에러 발생
에러 메시지: "TypeError: Cannot read property 'name' of undefined"

코드:
```javascript
async function fetchUser(id) {
  const response = fetch(`/api/users/${id}`);
  return response.json();
}

const user = fetchUser(123);
console.log(user.name);
```

기대 동작: 사용자 이름 출력
실제 동작: undefined 에러

문제를 찾고 수정된 코드를 제공해줘.
왜 그런 문제가 발생했는지도 설명해줘.
```

### 3.3.3 2.3 리팩토링

**템플릿:**
```
다음 코드를 리팩토링해줘.

현재 코드:
```[언어]
[코드]
```

개선 목표:
- [가독성/성능/유지보수성]
- [...]

원칙:
- [DRY/SOLID/Clean Code 등]
```

**실전 예시:**
```
다음 Python 코드를 리팩토링해줘.

현재 코드:
```python
def calc(a, b, c):
    if c == 'add':
        return a + b
    elif c == 'sub':
        return a - b
    elif c == 'mul':
        return a * b
    elif c == 'div':
        return a / b
```

개선 목표:
- 더 읽기 쉽고 확장 가능하게
- 타입 힌트 추가
- 에러 핸들링 추가
- 새로운 연산 추가하기 쉽게

원칙:
- Clean Code
- Strategy 패턴 고려
```

### 3.3.4 2.4 문서화

**템플릿:**
```
다음 [코드/API/함수]에 대한 문서를 작성해줘.

코드:
```[언어]
[코드]
```

문서 형식:
- [JSDoc/Sphinx/Javadoc]

포함 내용:
- 함수 설명
- 파라미터 설명
- 반환값 설명
- 사용 예시
- 주의사항
```

## 3.4 3. 기획/PM 분야 프롬프트

### 3.4.1 3.1 요구사항 정의

**템플릿:**
```
[제품/기능]: [...]
[사용자]: [...]
[문제점]: [...]

다음 형식으로 요구사항 문서를 작성해줘:
1. 배경 및 목적
2. 사용자 스토리 (As a... I want... So that...)
3. 기능 요구사항 (우선순위별)
4. 비기능 요구사항
5. 제약사항
6. 성공 지표
```

**실전 예시:**
```
제품: 온라인 화상 회의 툴
기능: 자동 회의록 작성
사용자: 회의가 많은 팀 리더와 PM
문제점: 회의 중 메모하느라 대화에 집중 못함

요구사항 문서를 작성해줘. 특히:
- 사용자 스토리 5개
- Must have vs Nice to have 구분
- 프라이버시 관련 제약사항 포함
- DAU, 사용률 등 측정 가능한 성공 지표
```

### 3.4.2 3.2 경쟁사 분석

**템플릿:**
```
[우리 제품]: [...]
[경쟁사]: [...]

다음 항목으로 경쟁사 분석표를 만들어줘:
- 핵심 기능 비교
- 가격 정책
- 타겟 고객
- 강점/약점
- 차별화 포인트
- 우리의 기회 요소
```

### 3.4.3 3.3 사용자 페르소나

**템플릿:**
```
[제품/서비스]: [...]
[타겟 시장]: [...]

다음 정보로 상세한 사용자 페르소나 3개를 만들어줘:
- 인구통계학적 정보
- 직업 및 경력
- 목표와 동기
- Pain Points
- 우리 제품 사용 시나리오
- 선호하는 커뮤니케이션 채널
```

## 3.6 5. 데이터 분석 프롬프트

### 3.6.1 5.1 데이터 분석 요청

**템플릿:**
```
[데이터 설명]: [...]
[분석 목적]: [...]

데이터셋:
[CSV 파일 업로드 또는 데이터 붙여넣기]

다음을 분석해줘:
1. 기초 통계 (평균, 중앙값, 표준편차 등)
2. 결측치 및 이상치 확인
3. [특정 변수] 간 상관관계
4. 시각화 (그래프 3-5개)
5. 비즈니스 인사이트 도출
```

**실전 예시:**
```
데이터: 최근 6개월 간 e-커머스 매출 데이터
분석 목적: 매출 증대 전략 수립

컬럼: 날짜, 제품명, 카테고리, 수량, 매출액, 고객ID

분석 요청:
1. 월별 매출 트렌드
2. 제품 카테고리별 성과
3. 리피트 고객 vs 신규 고객 비율
4. 매출 상위 20% 제품 분석
5. 다음 달 매출 예측
6. 실행 가능한 3가지 액션 아이템 제시

결과는 경영진 보고용으로 요약해줘.
```

## 3.7 6. 교육/학습 프롬프트

### 3.7.1 6.1 개념 설명 요청

**템플릿:**
```
[개념/주제]: [...]
[나의 배경지식]: [초보/중급/고급]
[선호하는 학습 방식]: [예시 중심/이론 중심]

다음 형식으로 설명해줘:
1. 한 문장 요약
2. 왜 중요한지
3. 핵심 개념 3-5가지
4. 실생활 예시 2-3개
5. 흔한 오해
6. 더 공부할 리소스
```

**실전 예시:**
```
개념: 머신러닝의 "과적합(Overfitting)"
배경지식: 프로그래밍은 할 줄 알지만 ML은 초보
학습 방식: 예시와 비유로 설명

특히 다음을 포함해줘:
- 일상 생활에 빗댄 비유
- 코드 예시 (Python)
- 실제로 과적합을 발견하는 방법
- 해결 방법 3가지
```

### 3.7.2 6.2 연습 문제 생성

**템플릿:**
```
[주제]: [...]
[난이도]: [...]
[문제 수]: [...]

다음 형식으로 문제를 만들어줘:
- 문제 설명
- 힌트 (선택적)
- 정답
- 해설
```

## 3.8 7. 번역 프롬프트

### 3.8.1 7.1 전문적 번역

**템플릿:**
```
다음 [언어]를 [언어]로 번역해줘.

맥락: [...]
톤: [격식/비격식/친근함/전문적]
독자: [...]

번역 시 주의사항:
- [문화적 뉘앙스/전문 용어/...]
- [...]

원문:
[텍스트]
```

**실전 예시:**
```
다음 영어 이메일을 한국어로 번역해줘.

맥락: 미국 고객사에게 보내는 프로젝트 지연 안내
톤: 정중하고 전문적
독자: VP 급 의사결정자

주의사항:
- "We apologize" 같은 표현을 한국 비즈니스 문화에 맞게
- 구체적인 날짜는 그대로 유지
- 미국식 완곡 표현을 한국식 명확한 표현으로

원문:
[이메일 텍스트]

번역 후, 어떤 부분을 왜 그렇게 번역했는지 설명도 해줘.
```

## 3.9 8. 업무 자동화 프롬프트

### 3.9.1 8.1 반복 작업 자동화

**템플릿:**
```
[반복 작업]: [...]
[현재 방식]: [...]
[사용 도구]: [Excel/Python/Google Apps Script/...]

이 작업을 자동화하는 방법을 알려줘:
1. 단계별 프로세스
2. 필요한 코드/스크립트
3. 설정 방법
4. 주의사항
```

**실전 예시:**
```
반복 작업: 매일 아침 여러 소스에서 데이터 수집 → Excel 정리 → 이메일 발송
현재 방식: 수동으로 복붙하고 정리 (30분 소요)
사용 가능 도구: Python, Google Sheets, Gmail

자동화하고 싶은 것:
1. API에서 데이터 가져오기
2. Google Sheets에 자동 입력
3. 간단한 차트 생성
4. 매일 오전 9시 자동으로 팀에게 이메일

초보자도 따라할 수 있게 단계별로 설명해줘.
```

## 3.10 9. 창의적 작업 프롬프트

### 3.10.1 9.1 브레인스토밍

**템플릿:**
```
[목표]: [...]
[제약사항]: [...]
[타겟]: [...]

다음 형식으로 아이디어를 제안해줘:
- 각 아이디어당 제목, 설명, 장점, 단점
- 실행 난이도 (쉬움/보통/어려움)
- 예상 효과

최소 10개 이상 제안해줘.
```

**실전 예시:**
```
목표: 신규 사용자 유입 늘리기
제약사항: 마케팅 예산 월 100만원 이하
타겟: 20-30대 직장인
제품: 업무 생산성 앱

다음 채널을 고려해서 아이디어 내줘:
- SNS (인스타, 유튜브, 틱톡)
- 콘텐츠 마케팅
- 바이럴 마케팅
- 파트너십

각 아이디어는:
- 구체적인 실행 방법
- 예상 비용
- 예상 결과 (정량적)
- 위험 요소
```

### 3.10.2 9.2 네이밍

**템플릿:**
```
[대상]: [제품/서비스/기능]
[핵심 가치]: [...]
[타겟 시장]: [...]
[느낌]: [...]

조건:
- 발음하기 쉬운
- 기억하기 쉬운
- 도메인 사용 가능 (선택)
- [언어] 뉘앙스 고려

20개 이상 제안해줘. 각각 설명도 함께.
```
