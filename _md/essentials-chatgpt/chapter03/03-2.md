## 3.1 고급 프롬프트 기법

### 3.1.1 1. Chain of Thought (CoT) - 단계별 사고

AI에게 단계별로 생각하도록 유도하면 더 정확한 답변을 얻을 수 있습니다.

**기본 프롬프트:**
```
Q: 72 ÷ 8 + 5 × 3 = ?
A: 24
(틀린 답변 가능성)
```

**CoT 적용:**
```
Q: 72 ÷ 8 + 5 × 3 = ?
단계별로 계산해줘:
1. 먼저 나눗셈과 곱셈을 계산
2. 그 다음 덧셈

A:
1단계: 72 ÷ 8 = 9, 5 × 3 = 15
2단계: 9 + 15 = 24
정답: 24
```

**실전 활용:**
```
나쁜 예:
"이 마케팅 전략이 좋은지 평가해줘"

좋은 예:
"이 마케팅 전략을 다음 순서로 평가해줘:
1. 타겟 고객 분석
2. 예산 대비 효과성
3. 실행 가능성
4. 위험 요소
5. 종합 평가 및 개선안"
```

### 3.1.2 2. Few-Shot Learning - 예시 제공

예시를 2-3개 제공하면 AI가 패턴을 학습합니다.

**Zero-Shot (예시 없음):**
```
다음 문장을 긍정/부정으로 분류해줘:
"이 제품 정말 별로네요"
```

**Few-Shot (예시 제공):**
```
다음과 같은 패턴으로 문장을 분류해줘:

예시:
"이 제품 너무 좋아요!" → 긍정
"돈 아까워요" → 부정
"배송이 빨라서 만족합니다" → 긍정

이제 다음 문장을 분류해줘:
"이 제품 정말 별로네요"
```

**실전 활용 - 글쓰기 스타일 학습:**
```
다음은 우리 회사 블로그의 스타일이야:

예시 1:
[과거에 잘 쓴 글 1 붙여넣기]

예시 2:
[과거에 잘 쓴 글 2 붙여넣기]

이 스타일로 다음 주제의 글을 써줘: [새로운 주제]
```

### 3.1.3 3. Role Prompting - 역할 부여

AI에게 특정 전문가 역할을 부여하면 더 전문적인 답변을 얻습니다.

**기본:**
```
"Python 코드를 최적화해줘"
```

**역할 부여:**
```
"너는 15년 경력의 시니어 Python 개발자야.
코드 성능 최적화 전문가로서,
다음 코드를 분석하고 개선해줘.
특히 시간 복잡도와 메모리 사용량을 고려해서."
```

**다양한 역할 예시:**
- 마케팅: "너는 실리콘밸리 스타트업의 Growth Hacker야"
- 기획: "너는 10년 경력의 Product Manager야"
- 디자인: "너는 Apple의 UI/UX 디자이너야"
- 교육: "너는 초등학생에게 설명하는 과학 선생님이야"

### 3.1.4 4. Prompt Chaining - 프롬프트 연결

복잡한 작업을 여러 단계로 나누어 진행합니다.

**한 번에 요청 (비효율):**
```
"제품 런칭 마케팅 캠페인 전체를 기획해줘"
→ 너무 광범위해서 품질 저하
```

**단계별로 나누기 (효율적):**
```
1단계: "타겟 고객 페르소나 3개 만들어줘"
   → 결과 검토

2단계: "위 페르소나들의 Pain Point 분석해줘"
   → 결과 검토

3단계: "각 Pain Point에 맞는 메시징 전략 제안해줘"
   → 결과 검토

4단계: "메시징 전략을 바탕으로 채널별 캠페인 기획해줘"
   → 최종 결과
```

### 3.1.5 5. Constraints - 제약사항 명시

명확한 제약사항을 주면 더 정확한 결과를 얻습니다.

**제약 없음:**
```
"신제품 출시 이메일 써줘"
→ 너무 길거나 톤이 안 맞을 수 있음
```

**제약 명시:**
```
"신제품 출시 이메일 써줘

제약사항:
- 길이: 200단어 이내
- 톤: 친근하고 설득력 있게
- 포함 필수: 할인 혜택, 마감일
- 제외: 과장된 표현, 전문 용어
- CTA 버튼 문구: 정확히 10자 이내"
```

### 3.1.6 6. Output Format - 출력 형식 지정

원하는 형식을 명확히 지정합니다.

**형식 미지정:**
```
"경쟁사 분석해줘"
→ 텍스트로 장황하게 설명
```

**형식 지정:**
```
"경쟁사 분석을 다음 형식의 표로 만들어줘:

| 항목 | 우리 | 경쟁사 A | 경쟁사 B |
|------|------|----------|----------|
| 가격 |      |          |          |
| 기능 |      |          |          |
| 강점 |      |          |          |
| 약점 |      |          |          |

표 아래에 3줄로 요약 추가"
```

**다양한 형식 지정:**
- JSON: "다음 형식의 JSON으로 출력해줘"
- 마크다운: "# 제목, ## 소제목 형식으로"
- 코드 블록: "```python 코드 블록 안에"
- 불릿 포인트: "- 형식의 리스트로"

### 3.1.7 7. Iterative Refinement - 반복 개선

첫 결과물을 바탕으로 점진적으로 개선합니다.

**1차 시도:**
```
"블로그 글 초안을 써줘: AI 도입 방법"
```

**2차 개선:**
```
"이 글의 도입부를 더 흥미롭게 바꿔줘.
통계나 질문으로 시작하면 좋겠어."
```

**3차 개선:**
```
"2번째 섹션에 실제 사례 2개 추가해줘"
```

**4차 개선:**
```
"전체적으로 톤을 더 친근하게 바꿔줘.
'해야 합니다' → '하면 좋아요' 스타일로"
```

### 3.1.8 8. Negative Prompting - 원하지 않는 것 명시

하지 말아야 할 것을 명확히 알려줍니다.

**포지티브만:**
```
"제품 소개 글 써줘"
→ 과장, 클리셰가 포함될 수 있음
```

**네거티브 추가:**
```
"제품 소개 글 써줘

포함하지 말 것:
- '혁신적', '획기적' 같은 과장된 표현
- '업계 최초', '세계 최고' 같은 검증 안 된 주장
- 클리셰 ('고객 만족 1위', '압도적 성능' 등)
- 기술 전문용어

대신:
- 구체적인 수치와 사실
- 실제 사용자 후기
- 명확한 비교 데이터"
```

### 3.1.9 9. Temperature & Creativity 조절

AI의 창의성 수준을 요청에 맞게 조절합니다.

**정확성 중요 (Temperature 낮게):**
```
"다음 계약서의 핵심 조항을 정확히 요약해줘.
절대 추측하거나 의견을 넣지 말고,
문서에 있는 내용만 사실 그대로 정리해줘."
```

**창의성 중요 (Temperature 높게):**
```
"신제품 이름 브레인스토밍해줘.
최대한 창의적이고 독특하게!
기존에 없던 조합, 신조어도 좋아.
30개 이상 다양하게 제안해줘."
```

## 3.2 실전 팁

### 3.2.1 1. 프롬프트 디버깅

결과가 만족스럽지 않을 때:

**문제 진단:**
```
1. 너무 모호했나? → 더 구체적으로
2. 맥락이 부족했나? → 배경 정보 추가
3. 너무 복잡했나? → 단계별로 나누기
4. 예시가 필요했나? → Few-shot 적용
5. 형식이 불분명했나? → 출력 형식 명시
```

**Before & After 예시:**

**Before (문제):**
```
"우리 제품 홍보 글 써줘"
→ 너무 일반적이고 브랜드 정체성 없음
```

**After (개선):**
```
역할: 너는 Z세대를 타겟하는 SNS 마케터야
제품: 친환경 텀블러
타겟: 20대 직장인, 환경에 관심 많음
톤: 친근하고 위트 있게
길이: 인스타그램 캐러셀 1장 (50자)
포함: 환경 보호 메시지 + 실용성
금지: 교훈적인 톤, 진부한 표현

5가지 버전을 만들어줘"
```

### 3.2.2 2. 메타 프롬프트

AI에게 더 좋은 프롬프트를 만들어 달라고 요청합니다.

**메타 프롬프트:**
```
"나는 [목표]를 하고 싶어.
더 좋은 결과를 얻으려면 어떤 프롬프트를 써야 할까?
개선된 프롬프트를 제안해줘."
```

**예시:**
```
사용자: "나는 블로그 글을 쓰고 싶어.
       더 좋은 결과를 얻으려면 어떤 프롬프트를 써야할까?"

AI: "다음과 같은 프롬프트를 추천합니다:

[역할]
너는 [산업] 분야의 전문 콘텐츠 작가야.

[맥락]
- 타겟 독자: [구체적 명시]
- 목적: [인지도/리드/교육 등]
- SEO 키워드: [주요 키워드]

[작업]
다음 주제로 블로그 글을 써줘: [주제]

[형식]
- 제목: 클릭을 유도하는 제목
- 서론: 독자의 문제/궁금증 제기
- 본문: 3-5개 섹션 (각 섹션 200-300자)
- 결론: 명확한 CTA
- 길이: 1,500-2,000자
- 톤: [친근한/전문적인/etc]"
```

### 3.2.3 3. 컨텍스트 관리

대화가 길어질 때 컨텍스트를 정리합니다.

**문제 상황:**
```
10번의 대화 후...
AI가 처음 논의한 내용을 잊어버림
```

**해결책:**
```
"지금까지 논의한 내용을 요약해줘:
1. [...]
2. [...]
3. [...]

이걸 바탕으로 다음 단계를 진행하자."
```

### 3.2.4 4. 비교 요청

여러 옵션을 비교해서 제시받습니다.

```
"다음 3가지 옵션을 비교 분석해줘:
- 옵션 A: [...]
- 옵션 B: [...]
- 옵션 C: [...]

비교 기준:
1. 비용
2. 실행 난이도
3. 예상 효과
4. 위험도

표로 정리하고, 추천안과 이유를 알려줘."
```

### 3.2.5 5. 역질문 유도

AI가 먼저 질문하게 만듭니다.

```
"나는 [목표]를 하려고 해.
더 좋은 제안을 하기 위해 나에게 질문 5개를 해줘.
내가 답변하면 그걸 바탕으로 제안해줘."
```

## 3.3 도구별 특화 팁

### 3.3.1 ChatGPT

- **Custom Instructions**: 매번 역할/톤 설정 불필요
- **Code Interpreter**: 데이터 파일 업로드 가능
- **DALL-E 통합**: 이미지 생성 즉시 가능

### 3.3.2 Claude

- **긴 문서**: 100K+ 토큰, 책 전체 분석 가능
- **Artifacts**: 코드/문서를 별도 창으로 관리
- **정확성**: Hallucination 적음

### 3.3.3 Gemini

- **실시간 검색**: 최신 정보 자동 반영
- **YouTube 통합**: 영상 URL 붙여넣기 가능
- **Google Workspace**: Docs, Sheets 연동

## 3.4 피해야 할 실수

### 3.4.1 1. 너무 짧은 프롬프트

```
❌ "코드 짜줘"
✅ "Python으로 CSV를 읽어서 그래프 그리는 코드 짜줘.
   pandas와 matplotlib 사용, 주석 포함"
```

### 3.4.2 2. 맥락 없이 요청

```
❌ "이거 어떻게 생각해?"
✅ "우리 회사는 B2B SaaS 스타트업이야.
   다음 마케팅 전략에 대한 의견을 듣고 싶어: [전략]"
```

### 3.4.3 3. 한 번에 너무 많은 요청

```
❌ "제품 기획, 디자인, 개발, 마케팅 전략 다 알려줘"
✅ "먼저 제품 기획부터 시작하자.
   타겟 고객 페르소나 3개를 만들어줘."
```

### 3.4.4 4. 검증 없이 신뢰

```
❌ AI 답변을 그대로 사용
✅ 중요한 정보는 반드시 검증
   "이 통계의 출처를 알려줘"
   "이 법률 정보가 정확한지 확인이 필요해"
```
