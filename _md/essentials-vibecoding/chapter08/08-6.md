이 장에서는 비개발자가 백엔드 서비스를 배포하는 방법을 다룹니다. Django 프로젝트를 Render에 배포하는 실습을 통해 배포 과정을 이해합니다.

<highlight>처음 웹 서비스를 만들고 배포해본 사람에게는 큰 재미와 동시에 막막함을 함께 느낍니다.</highlight> 더 무엇을 공부해야 할지, 어디에 배포해야 할지, 모든게 다 어렵고 힘들게 느껴지기 때문이죠. 만약 처음 입문하신 분이라면 무작정 4 ~ 5번 반복을 해보시길 권해드립니다. 여기서 자연스럽게 습득이 되는 부분이 많기 때무입니다.

# 1. 무료인 서비스는 없다

우리가 앞으로 개발해서 배포해야 하는 서비스는 보통 '연산' 기능을 포함합니다. 이러한 '연산' 기능을 제공하는 서버는 24시간 켜져 있어야 하며, 이를 위해서는 서버 비용이 발생합니다. 따라서 완전히 무료로 서비스를 배포하는 것은 현실적으로 어렵습니다.

집에 있는 컴퓨터를 활용하더라도, NAS와 같은 장비를 사용하더라도 전기세가 발생되며 안정적인 네트워크 구축도 필요합니다.

# 2. 쉬운 배포 서비스

일시적으로나마 무료로 백엔드 서비스를 배포할 수 있는 플랫폼 들이 있습니다. 대표적으로 Render가 있습니다. 

* 링크: https://render.com/

그러면 우리는 이렇게 반문해볼 수 있습니다. '블로그 처럼 쉬운 백엔드 서비스 배포는 왜 없을까?'라는 것이죠. 이는 몇 가지 이유가 있습니다.

1. **보안 문제**: 배포된 서비스는 외부에 노출되기 때문에 보안이 매우 중요합니다. 비개발자가 쉽게 사용할 수 있는 배포 서비스는 보안 취약점이 발생할 가능성이 높습니다. 예를 들어, 여러분의 아이디와 패스워드가 노출되면 누군가 여러분 서버에서 비트코인 채굴을 하고 비용을 여러분이 부담해야 할 수도 있습니다. 이러한 보안 문제가 생겼을 때 대부분의 클라우드 서비스는 관대하지 않습니다.

2. **복잡성 문제**: 배포 과정은 단순히 코드를 올리는 것 이상으로, 서버 설정, 데이터베이스 연결, 보안 설정 등 다양한 요소를 포함합니다. 이러한 복잡성 때문에 비개발자가 쉽게 접근하기 어렵습니다.

물론 위 2가지 말고도 더 많은 이유가 있지만, 가장 큰 이유는 위 2가지입니다. 그럼에도 Render는 매우 빠르고 간단한 배포 경험을 제공합니다.

# 3. 개발된 서비스 배포하기

앞서 개발한 블로그를 가지고 배포를 해보도록 하겠습니다. 만약 앞서 실습한 파일이 없다면 미리 준비된 코드를 다운로드 받아서 진행해도 무방합니다.

* 링크: https://github.com/paullabkorea/vibe_django_test

다운로드를 받으시고 한 번 실행해보세요. 파이썬이 설치되어 있지 않다면 설치부터 진행해야 합니다. 파이썬은 아래 페이지에서 다운로드 받을 수 있습니다.

* 링크: https://www.python.org/

이 문서에도 실행 방법이 상세 정리되어 있습니다. `사용법.md`에 정리된 문서대로 실행해보겠습니다. 나머지 실행 단계가 모두 수행되었다는 전제 하에 터미널에서 아래 코드를 입력하면 서버가 구동됩니다.

```
python manage.py runserver
```

`사용법.md`를 참고해서 아래 URL에 접속합니다.

```
http://127.0.0.1:8000/
```

로그인이 완료되면 영상을 올려보도록 합니다. http://127.0.0.1:8000/admin 로 접속하여 Videos를 눌러 비디오를 올립니다. 새 게시물을 올렸다면 http://127.0.0.1:8000/ 페이지에 다시 접속을 해보면 올라온 영상을 볼 수 있습니다. 영상 재생도 잘 됩니다.

# 4. 이미 개발된 서비스를 수정하기

이렇게 개발이 되어 있는 서비스를 바로 Render에 올릴 수도 있지만, 우리는 여기에 간단한 게시판을 한 번 추가해보도록 하겠습니다. 클로드 코드에게 다음과 같이 요청할 수 있습니다.

```
간단한 게시판을 하나 추가해줘.
```

다만, 이렇게 하지 않고 아래처럼 앞서 배운 것을 활용해서 URL 설계, DB 설계 후 다음과 같은 프롬프트로 요청하는 것이 더 좋습니다.

```
나는 간단한 게시판을 추가로 만들려고 해.

[주요 기능]
- 게시글 작성, 수정, 삭제
- 게시글 목록 보기
- 게시글 상세 보기

[URL 설계]
[앞서 진행한 URL 설계 내용 + 게시판 URL 설계 삽입]

[DB 설계]
[앞서 진행한 DB 설계 내용 + 게시판 DB 설계 삽입]
Django 기반으로 이 기능을 구현해줘.
```
