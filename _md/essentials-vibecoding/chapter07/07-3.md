# 7.3 서비스 기획

# 1. 바이브 코딩을 위한 서비스 기획 접근법

## 1.1 바이브 코딩을 위한 서비스 기획 접근법

전통적인 서비스 기획은 완벽한 계획을 세우고 단계적으로 실행하는 것을 중시했지만, 바이브 코딩에서는 빠른 실험과 반복적 개선이 더욱 중요합니다. 이제 Claude Code를 이용하면 아이디어를 빠르게 프로토타입으로 만들어 시장의 반응을 확인할 수 있기 때문에, 기획 단계에서는 핵심 가설을 명확히 정의하고 이를 검증하는 데 집중해야 합니다.

### 1.1.1. 문제 정의와 솔루션 가설

성공적인 서비스는 명확한 문제에서 출발합니다. 하지만 단순히 문제를 나열하는 것이 아니라, 타겟 사용자가 누구이며 그들이 어떤 상황에서 어떤 어려움을 겪고 있는지 구체적으로 파악해야 합니다. 이러한 문제 정의를 바탕으로 솔루션 가설을 세우면, Claude Code와 함께 개발할 때도 일관된 방향성을 유지할 수 있습니다.

**문제 정의 템플릿**

```
타겟 사용자: [구체적인 사용자 그룹]
현재 상황: [사용자가 겪고 있는 문제]
원하는 결과: [사용자가 달성하고자 하는 목표]
장애물: [목표 달성을 방해하는 요소들]
```

**솔루션 가설**

```
만약 우리가 [솔루션]을 제공한다면,
[타겟 사용자]는 [기대하는 행동]을 할 것이고,
이를 통해 [비즈니스 목표]를 달성할 수 있을 것이다.
```

### 1.1.2. 사용자 여정 맵핑

사용자 여정을 단계별로 세분화하면 Claude Code로 개발할 때 각 단계별로 필요한 기능과 화면을 체계적으로 구현할 수 있습니다. 특히 각 단계에서 사용자가 이탈할 수 있는 지점을 미리 파악하고 이를 방지하는 기능을 설계하는 것이 중요합니다.

```
1. 인지 (Awareness)
   - 사용자가 서비스를 어떻게 알게 되는가?
   - 랜딩 페이지, SEO, 소셜미디어

2. 관심 (Interest)
   - 사용자의 관심을 어떻게 유지하는가?
   - 서비스 소개, 데모, 무료 체험

3. 고려 (Consideration)
   - 사용자가 선택을 위해 필요한 정보는?
   - 기능 비교, 가격 정보, 후기

4. 구매/가입 (Purchase)
   - 전환율을 높이는 요소는?
   - 간편 가입, 결제 시스템, 인센티브

5. 사용 (Usage)
   - 핵심 가치를 빠르게 경험하게 하려면?
   - 온보딩, 튜토리얼, 첫 성공 경험

6. 유지 (Retention)
   - 지속적 사용을 유도하는 방법은?
   - 알림, 커뮤니티, 개인화

7. 추천 (Advocacy)
   - 사용자가 다른 이에게 추천하게 하려면?
   - 추천 시스템, 소셜 기능, 인센티브
```

### 1.1.3. 핵심 메트릭 정의

서비스의 성공을 측정할 수 있는 지표를 명확히 정의합니다. 바이브 코딩에서는 빠른 개발과 배포가 가능하므로, 실제 데이터를 통해 지표를 모니터링하고 개선점을 찾는 것이 중요합니다. 또한 Claude Code로 분석 대시보드를 구축하여 이러한 지표들을 실시간으로 추적할 수 있습니다.

개발을 할 때에는 이러한 지표를 추적하고 관리할 수 있는 관리자 페이지를 설계할 수 있도록 요구사항 명세에 우선순위를 높여 가져가는 것이 중요합니다.

**AARRR 모델 적용**

```
Acquisition (고객 획득)
- 회원가입 수
- 트래픽 소스별 전환율
- CAC (Customer Acquisition Cost)

Activation (활성화)
- 첫 주요 액션 완료율
- 온보딩 완료율
- Time to Value (가치 인식까지 걸리는 시간)

Retention (고객 유지)
- DAU/MAU 비율
- Cohort 분석
- Churn Rate

Revenue (수익)
- ARPU (Average Revenue Per User)
- LTV (Lifetime Value)
- 결제 전환율

Referral (추천)
- 추천율
- 바이럴 계수
- NPS (Net Promoter Score)
```

# 2. MVP(Minimum Viable Product) 정의

Claude Code를 사용하면 복잡한 기능도 빠르게 구현할 수 있지만, 그렇다고 해서 처음부터 모든 기능을 포함해서는 안 됩니다. 오히려 핵심 가치를 전달할 수 있는 최소한의 기능으로 시작하여 사용자 피드백을 바탕으로 점진적으로 확장하는 것이 효과적입니다.

### 2.1 핵심 기능 (Core Features)

```
1순위: 사용자가 핵심 가치를 경험할 수 있는 최소 기능
- 사용자 가입/로그인
- 핵심 서비스 1개 기능
- 기본적인 UI/UX

2순위: 사용성 개선을 위한 기능
- 사용자 프로필 관리
- 검색 및 필터링
- 알림 시스템

3순위: 비즈니스 모델 구현
- 결제 시스템
- 구독 관리
- 분석 및 리포팅
```

### 2.2 기술적 제약사항 고려

```
개발 리소스: 1인 개발, Claude Code 활용
개발 기간: 4주 이내
예산: 클라우드 서비스 월 $100 이하
확장성: 사용자 1000명 까지 지원
```
