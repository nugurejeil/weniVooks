# 5. 생성형 AI 활용 브랜드 디자인

/목차

# **1. 이름 상표 검색**

로고 디자인을 시작하기 전에 가장 먼저 해야 할 중요한 단계가 있습니다. 우리가 사용할 서비스 이름이 이미 다른 곳에서 상표로 등록되어 있는지 확인하는 과정입니다.

## **1.1 왜 중요할까요?**

이미 등록된 상표를 사용하면 법적인 문제가 발생할 수 있습니다. 시간과 노력을 들여 로고를 만들었는데 사용하지 못하게 된다면 너무 아쉽겠죠? 미리 확인하는 것이 좋습니다.

## **1.2 어디서 검색하나요?**

- 국내에서는 특허청 '키프리스(KIPRIS)' 웹사이트에서 상표를 검색할 수 있습니다. 간단하게 서비스 이름을 검색하여 유사한 상표가 있는지 확인해 보시면 됩니다.[https://www.kipris.or.kr/khome/main.do](https://www.kipris.or.kr/khome/main.do)

- 또한 Google에서 내가 기획한 서비스의 이름을 검색해 보고 유사한 서비스가 이미 있는지 살펴보는 것도 좋은 방법입니다.

# **2. 유사 서비스 레퍼런스 분석**

우리의 서비스와 비슷한 다른 서비스들은 어떤 로고를 사용하고 있는지 살펴보는 시간입니다.

> 예시: 쇼핑 앱을 만들 예정이라면 → 지마켓, 지그재그, 다나와 등의 로고를 살펴보기
> 

## **2.1 벤치마킹의 가치**

성공적인 서비스들이 어떤 로고 디자인을 가지고 있는지 분석하면 우리 로고의 방향성을 잡는 데 도움이 됩니다.

## **2.2 차별점 찾기**

유사한 서비스들 속에서 우리 로고가 돋보이려면 어떤 독특한 요소를 넣을 수 있을지 고민해 볼 수 있습니다.

# **3. 로고 디자인의 종류 이해하기**

로고는 크게 몇 가지 형태로 나눌 수 있습니다. 우리 서비스에 어떤 형태가 가장 어울릴지 생각해 보시면 좋습니다.

- **텍스트형 (워드마크)**: 서비스 이름을 글자 자체로 디자인한 로고
- **심볼형 (로고마크)**: 서비스 특징을 상징하는 그림이나 아이콘 로고
- **앰블럼형**: 텍스트와 심볼을 함께 사용하는 로고
- **캐릭터형**: 동물이나 사물을 의인화한 형태의 로고

# **4. 로고 디자인 기본 지식**

## **4.1 벡터와 비트맵의 차이**

디자인에서 이미지를 표현하는 방식을 이해하면 더 좋은 로고를 만들 수 있습니다.

### **4.1.1 비트맵(Bitmap) - 점으로 그린 그림**

수많은 작은 점(픽셀)들이 모여서 만든 이미지입니다. 사진이나 스크린샷 같은 것들이죠. 확대하면 픽셀이 보여서 깨져 보일 수 있습니다.

### **4.1.2 벡터(Vector) - 수학으로 그린 그림**

점과 점을 연결한 선으로 그리는 방식입니다. 쉽게 말해 컴퓨터에게 "이 점에서 저 점까지 이런 곡선으로 그려줘"라고 명령하는 것이죠. 아무리 확대해도 깨지지 않습니다.

![5.4.1-3](/images/essentials-vibecoding/chapter05/5.png)

5.4.1-3

|  | 벡터(Vector) | 비트맵(Bitmap) |
| --- | --- | --- |
| 장점 | 확대하거나 축소해도 해상도에 영향을 끼치지 않는다. | 정교하고 다양한 색상을 가진 이미지를 만들 수 있다. |
| 단점 | 정교한 이미지, 그래픽 표현에는 적합하지 않음 | - 이미지가 깨질 수 있음
- 대형 출력물 제작 시 용량이 너무 커질 수 있음 |
| 형식/확장자 | AI, EPS, SVG | JPG, PNG, BMP, GIF |
| 적합한 사용 용도 | 로고, 아이콘, 일러스트 등 | 사진, 이미지, 그래픽 등 |

## **4.2 PNG와 SVG 이해하기**

- **PNG**: 배경이 투명한 비트맵 이미지 (웹에서 많이 사용)
- **SVG**: 벡터 데이터 이미지 (로고에 최적)

## **6.2 이미지를 벡터(SVG)로 변환하기**

생성된 PNG 이미지를 SVG로 변환하면 더 활용도가 높아집니다.

**변환 도구 추천**

Convertio([https://convertio.co/kr/](https://convertio.co/kr/)) 같은 무료 변환 서비스를 사용하면 쉽게 변환할 수 있습니다.

**SVG로 변환하면 좋은 점**

- 색상 변경이 자유로움
- 애니메이션 적용 가능
- 어떤 크기로도 선명하게 표시

# **7. 내 서비스에 최종 적용하기**

이제 모든 준비가 끝났습니다. 생성형 AI로 만든 로고를 여러분의 서비스에 적용해봅시다.

**Claude Code 활용 프롬프트**

```jsx
(로고 파일을 프로젝트 폴더에 넣은 뒤)
logo.svg 파일을 헤더 좌측에 추가해줘
크기는 height 40px로 설정하고 클릭하면 홈으로 가도록 링크도 연결해줘
```

# **8. 다양한 곳에 활용하기**

로고를 만들었다면 여러 곳에 활용할 수 있습니다.

**활용 예시**

- 명함 디자인
- SNS 프로필 이미지
- OG/메타 이미지 (링크 공유 시 나타나는 썸네일)
- 이메일 서명
- 프레젠테이션 템플릿

---

로고는 서비스의 첫인상을 결정하는 중요한 요소입니다. 완벽한 로고를 한 번에 만들기는 어렵지만 AI의 도움을 받아 여러 버전을 빠르게 시도해보면서 가장 마음에 드는 것을 찾아가시면 좋습니다.
