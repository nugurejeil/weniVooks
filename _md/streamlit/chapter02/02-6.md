Streamlit은 기본적으로 단일 페이지 기반의 앱을 지원하지만, 다양한 레이아웃 구성 기능을 제공하여 UI를 효율적으로 관리하고 구조화할 수 있습니다. 예를 들어, 여러 개의 컬럼을 나누고 각 컬럼에 요소를 배치하거나, 특정 요소들을 그룹화하여 스타일링과 레이아웃 관리를 손쉽게 할 수 있습니다.

<br>

이러한 레이아웃 기능은 애플리케이션의 외관을 정돈되고 직관적으로 구성할 수 있게 해주며, 사용자가 보다 쉽고 편리하게 이용할 수 있도록 사용자 경험(UX)을 개선하는데 유용합니다. 또한, 화면에 다양한 요소를 배치하는 여러 옵션을 제공하여, 개발자가 원하는 방식으로 유연하게 디자인할 수 있도록 지원합니다.

<br>

# st.columns

`st.columns`는 화면에 여러 개의 열(column) 을 나란히 배치할 수 있도록 컨테이너를 생성합니다. 이를 통해 각 열에 원하는 요소를 삽입하여 수평 방향의 레이아웃을 구성할 수 있습니다. 생성된 각 열은 컨테이너 객체이며, with 표기법(기본값) 또는 반환된 객체에서 직접 객체 메서드를 호출하여 요소를 추가할 수 있습니다.

열의 너비 비율과 배치 순서는 함수의 인자를 통해 조정할 수 있습니다. 열은 최대 1단계까지만 중첩이 가능하며, 다른 열 안에 배치할 수 있습니다. `st.columns`는 사이드바의 다른 열 안에서는 사용할 수 없습니다. `st.columns`는 메인 페이지 영역에서만 작동합니다.

```python
st.columns(spec, *, gap="small", vertical_alignment="top", border=False)
```

<br>

**매개변수**

- `spec`(int, Iterable of numbers) : 레이아웃을 구성하기 위해 삽입할 열의 수와 너비를 구성합니다.
  - int : 정수값만큼의 개수의 열이 한 행에 만들어집니다. 이때, 모든 열의 너비는 동일합니다.
  - iterable of numbers, int, float : 각각의 열의 너비 비율(상대 너비)을 지정할 수 있습니다.
    ex1) [0.7, 0.3]은 첫 번째 열이 사용 가능한 열의 70%를 차지하고 두 번째 열이 30%를 차지하는 두 개의 열을 만듭니다.
    ex2) [1, 2, 3]은 두 번째 열은 첫 번째 열 너비의 2배, 세 번째 열은 그 너비의 3배인 세 개의 열을 만듭니다.
- `gap`("small", "medium", "large", Keyword only) : 열 사이의 간격을 조정합니다.
  - "small"(기본값)
  - "medium"
  - "large"
- `vertical_alignment`("top", "center", "bottom", Keyword only) : 열 내부 콘텐츠를 세로 정렬합니다.
  - "top"(기본값)
  - "center"
  - "bottom"
- `border`(bool, Keyword only) : 열 컨테이너 주위에 테두리를 표시할지 여부입니다.
  - True : 각 열 주위에 테두리가 표시됩니다.
  - False(기본값) : 테두리가 표시되지 않습니다.

<br>

**반환값**

- list of containers : 컨테이너 객체 리스트입니다.

<br>

**표기법**

`st.columns`는 객체 표기법과 with 표기법을 사용하여 요소를 추가할 수 있습니다.

- `st.columns` 객체 표기법
  ```python
  st.columns(spec)[열 인덱스].widget_name
  ```
- with 표기법
  ```python
  with st.columns(spec)[열 인덱스]:
      st.widget_name
  ```

<br>

**예시 1**

다음 예시는 `st.columns`를 활용해 한 행에 두 개의 열을 구성한 예시입니다. st.columns([3, 1])은 3:1 비율로 두 개의 열을 생성하며, 첫 번째 열(col1)은 넓고 두 번째 열(col2)은 상대적으로 좁은 공간을 가집니다. 이러한 비율 설정은 차트와 표를 나란히 배치하거나, 강조할 콘텐츠를 넓은 영역에 배치할 때 유용합니다. 생성된 열은 각각 컨테이너 객체로 반환하며, 요소를 추가할 때는 with 문을 사용하거나 반환된 객체에서 컨테이너 메서드를 직접 호출하여 명령을 실행할 수 있습니다.

예시에서는 각 열에 제목과 이미지가 하나씩 배치되어 있으며, col1에는 고양이 이미지, col2에는 강아지 이미지가 표시됩니다. 또한, 시각적으로 구분을 돕기 위해 간단한 CSS를 활용해 각 열의 배경에 연한 파란색(lightblue)을 적용함으로써 컬럼 영역을 시각적으로 구분할 수 있도록 구성했습니다.

```python
import streamlit as st

col1, col2 = st.columns([3,1])

# 컬럼 영역 구분을 위한 css 코드 추가 (파란 가로선)
st.markdown(
    """
    <style>
    .custom-column {
        background-color: lightblue;
        padding: 5px;
    }
    </style>
    """,
    unsafe_allow_html=True
)

col1.subheader("고양이")
col1.markdown('<div class="custom-column">', unsafe_allow_html=True)
col1.image("https://static.streamlit.io/examples/cat.jpg")

col2.subheader("강아지")
col2.markdown('<div class="custom-column">', unsafe_allow_html=True)
col2.image("https://static.streamlit.io/examples/dog.jpg")
```

**예시 2**

다음 예시는 `st.columns`과 with 문을 활용하여 한 행에 세 개의 열을 구성한 코드입니다. 각 열은 col1, col2, col3으로 나뉘며, with 블록 안에서 각각의 열에 제목과 이미지를 삽입합니다. with 문을 사용하여 열에 모든 요소를 삽입할 수 있습니다. 이처럼 컨텍스트 관리 방식을 사용하면 열마다 독립적으로 요소를 구성할 수 있어 가독성과 유지보수 측면에서 유리합니다. 또한, CSS를 활용하여 각 열에 연한 파란색(lightblue) 배경을 적용함으로써 시각적으로 영역을 구분할 수 있도록 구성했습니다. 결과적으로 col1에는 고양이, col2에는 강아지, col3에는 부엉이 이미지가 각각 표시되며, 이미지 갤러리나 항목 비교 레이아웃을 구성할 때 유용한 패턴입니다.

```python
import streamlit as st

col1, col2, col3 = st.columns(3)

# 컬럼 영역 구분을 위한 css 코드 추가 (파란 가로선)
st.markdown(
    """
    <style>
    .custom-column {
        background-color: lightblue;
        padding: 5px;
    }
    </style>
    """,
    unsafe_allow_html=True
)

with col1:
    st.header("고양이")
    st.markdown('<div class="custom-column">', unsafe_allow_html=True)
    st.image("https://static.streamlit.io/examples/cat.jpg")

with col2:
    st.header("강아지")
    st.markdown('<div class="custom-column">', unsafe_allow_html=True)
    st.image("https://static.streamlit.io/examples/dog.jpg")

with col3:
    st.header("부엉이")
    st.markdown('<div class="custom-column">', unsafe_allow_html=True)
    st.image("https://static.streamlit.io/examples/owl.jpg")
```

**예시 3**

다음 예시는 2개의 열을 만들어 각 열에 이미지와 텍스트를 배치한 예시입니다. `gap`을 사용하여 이미지와 텍스트의 간격이 “large”로 조절됩니다.

```python
import streamlit as st
from PIL import Image

col1, col2 = st.columns(2, gap="large")

# 컬럼 영역 구분을 위한 css 코드 추가 (파란 가로선)
st.markdown(
    """
    <style>
    .custom-column {
        background-color: lightblue;
        padding: 5px;
    }
    </style>
    """,
    unsafe_allow_html=True
)

with col1:
    st.subheader("column 1")
    image = Image.open("profile.png")
    st.markdown('<div class="custom-column">', unsafe_allow_html=True)
    st.image(image, caption="profile", width=200)

with col2:
    st.subheader("column 2")
    st.markdown('<div class="custom-column">', unsafe_allow_html=True)
    st.header("Jane")
    st.write("Hobby : Soccer")
    st.write("Nice to meet you.")
```

**예시 7**

다음 예시는 `st.text_input` 위젯의 다양한 옵션을 활용하여 레이블 표시 여부, 비활성화 설정, 도움말 텍스트를 정의합니다. 좌측 열에서는 체크박스와 라디오 버튼, 텍스트 입력을 통해 사용자로부터 값을 입력받고, 우측 열에서는 값들에 따라 동적으로 텍스트 입력 위젯의 상태가 변경됩니다. 이처럼 상호작용을 통해 위젯의 동작을 실시간으로 확인할 수 있어 위젯 속성 학습에 효과적입니다.

```python
import streamlit as st

col1, col2 = st.columns(2)

with col1:
    disabled = st.checkbox("Disable text input widget", key="disabled")
    visibility = st.radio(
        "Set text input label visibility 👉",
        key="visibility",
        options=["visible", "hidden", "collapsed"],
    )
    placeholder = st.text_input(
        "Placeholder for the other text input widget",
        "This is a placeholder",
        key="placeholder",
    )

with col2:
    text_input = st.text_input(
        "Enter some text 👇",
        label_visibility=visibility,
        disabled=disabled,
        placeholder=placeholder,
    )

    if text_input:
        st.write("You entered: ", text_input)
```

# st.container

`st.container`는 화면에 별도의 시각적 표시(테두리, 배경색, 여백 등) 없이 사용할 수 있는 여러 요소를 묶어주는 컨테이너를 생성합니다. `st.container`를 사용하면 요소들을 그룹화하고 순서대로 생성할 수 있으며, 필요한 시점에 표시하거나 숨길 수 있습니다. 예를 들어 `st.container`로 여러 요소를 묶으면, 컨테이너 안의 내용들을 나중에 조건적으로 보여주거나 반복적으로 배치할 수 있지만, 사용자 입장에서는 특별한 시각적 변화 없이 자연스럽게 내용이 나타나는 것처럼 보입니다.

```python
st.container(*, height=None, border=None, key=None)
```

<br>

**매개변수**

- `height`(int, None, Keyword only) : 컨테이너의 원하는 높이를 픽셀 단위로 조절할 때 사용됩니다.
  - None(기본값)인 경우 컨테이너는 콘텐츠에 맞게 설정합니다. 만약, 높이가 고정된 경우 큰 콘텐츠에 스크롤이 활성화되고 컨테이너 주위에 회색 테두리가 표시되어 스크롤 표면과 앱의 나머지 부분을 시각적으로 구분합니다.
  - 스크롤이 있는 컨테이너는 가급적 사용하지 않는 것을 권장드립니다. 사용하는 경우 높이를 작게(500픽셀 이하) 유지하세요. 그렇지 않으면 컨테이너의 스크롤 표면이 모바일 디바이스에서 화면의 대부분을 차지하여 앱의 나머지 부분을 스크롤하기 어려울 수 있습니다.
- `border` (bool, None, Keyword only) : 컨테이너 주위에 테두리를 표시할지 여부입니다. None(기본값)을 선택하면 컨테이너가 고정 높이로 설정된 경우 테두리가 표시되고 그렇지 않은 경우 테두리가 표시되지 않습니다.
- `key`(str, None, Keyword only) : 컨테이너에 안정적인 ID를 부여하기 위한 선택적 문자열입니다. 또한 `key`를 제공하면 접두사가 st-key-로 시작되는 CSS 클래스 이름으로 사용됩니다.

<br>

**표기법**

`st.container` 객체 표기법과 with 표기법을 사용하여 요소를 추가할 수 있습니다. 이때, `st.container` 안에 다른 `st.container`를 넣을 수 없습니다.

- `st.container` 객체 표기법
  ```python
  st.container().widget_name
  ```
- with 표기법
  ```python
  with st.container():
      st.widget_name
  ```

<br>

**예시 1**

다음 예시는 임의의 데이터 생성된 데이터를 with 문을 사용하여 `st.container`으로 시각화하는 예시입니다.

```python
import streamlit as st

with st.container():
    st.write("이것은 컨테이너 내부입니다")
    st.image("https://static.streamlit.io/examples/cat.jpg")

st.write("이것은 컨테이너 외부입니다")
```

# st.empty

`st.empty`는 단일 요소 컨테이너를 생성하며, 해당 공간은 언제든지 요소를 제거하거나 여러 요소를 한 번에 교체할 수 있는 하위 다중 요소 컨테이너로 활용됩니다. 주로 조건에 따라 동적인 내용을 표시할 때 유용합니다.

```python
st.empty()
```

<br>

**매개변수**

`st.empty`는 매개변수가 없습니다.

<br>

**표기법**

`st.empty`는 객체 표기법과 with 표기법을 사용하여 요소를 추가/교체/제거할 수 있습니다.

- `st.empty` 객체 표기법
  ```python
  st.empty().widget_name
  ```
- with 표기법
  ```python
  with st.empty():
      st.widget_name
  ```

<br>

**예시 1**

# st.expander

`st.expander`는 사용자가 확장(펼침)하거나 축소(접음)할 수 있는 패널(Expander) 컨테이너를 생성해 여러 요소를 담는데 사용합니다. 접었을 때 표시되는 것은 제공된 레이블뿐입니다.

```sql
st.expander(label, expanded=False, *, icon=None)
```

:::div{.callout}

`st.expander` 안에 다른 `st.expander`를 넣을 수 없습니다.

:::

<br>

**매개변수**

- `label`(str) : 확장기의 헤더로 사용할 문자열로 접었을 때 표시되는 레이블입니다. 일부 마크다운 문법을 지원합니다. 이미지는 아이콘처럼 표시되며 최대 높이는 글꼴 높이와 같습니다. 지원되지 않는 마크다운 요소는 래핑이 해제되어 해당 하위 요소(텍스트 콘텐츠)만 렌더링됩니다. 지원되지 않는 요소는 백슬래시 이스케이프를 사용하여 리터럴 문자로 표시합니다. ex) 1\. 정렬된 목록이 아님
- `expanded`(bool) : 기본값은 False(축소, 접힌 상태)이며, True인 경우 확장 상태로 초기화합니다.
- `icon`(str, None, Keyword only) : 확장자 레이블 옆에 표시할 이모티콘 또는 아이콘입니다.
  - None(기본값) : 아이콘이 표시되지 않습니다.
  - 한 글자 이모티콘을 설정할 수 있으며, 이모티콘 단축 코드는 지원되지 않습니다.
    ex) icon="🚨", icon="🔥"
  - 소재 기호 라이브러리의 아이콘(둥근 스타일)으로 ":소재/아이콘*이름:" 형식의 아이콘(여기서 "아이콘*이름"은 snake 표기로 된 아이콘 이름).
    ex) icon=":material/thumb_up:"은 👍🏻 아이콘을 표시합니다.
    https://fonts.google.com/icons?icon.set=Material+Symbols&icon.style=Rounded 글꼴 라이브러리에서 추가 아이콘을 찾을 수 있습니다.

<br>

**표기법**

`st.expander` 컨테이너에 요소를 추가하려면 with 표기법을 사용하거나 개체에서 직접 메서드를 호출하면 됩니다.

- `st.expander` 객체 표기법
  ```python
  st.expander(label).widget_name
  ```
- with 표기법
  ```python
  with st.expander(label):
      st.widget_name
  ```

<br>

# st.form

`st.form`은 다른 요소와 위젯을 시각적으로 그룹화하고 'Submit' 버튼과 함께 요소를 일괄 처리하는 컨테이너입니다. 폼의 ‘Submit’ 버튼을 누르면 양식 내의 모든 위젯 값이 일괄적으로 Streamlit으로 전송됩니다.

- 모든 폼에는 `st.form_submit_button`이 포함되어야 합니다.
- `st.button` 및 `st.download_button`은 폼에 추가할 수 없습니다.
- 폼은 앱의 `st.sidebar`, `st.column` 등 어디에나 표시할 수 있지만 다른 폼 안에 삽입할 수는 없습니다.
- 폼 내에서 콜백 함수를 가질 수 있는 유일한 위젯은 `st.form_submit_button`입니다.

```python
st.form(key, clear_on_submit=False, *, enter_to_submit=True, border=True)
```

<br>

**매개변수**

- `key`(str) : 폼을 식별하는 문자열입니다. 각 폼에는 고유한 키가 있어야 하며, `key`는 인터페이스에서 사용자에게 표시되지 않습니다.
- `clear_on_submit`(bool) : True로 설정하면 사용자가 제출 버튼을 누른 후 양식 내의 모든 위젯이 기본값으로 재설정됩니다. 기본값은 False입니다. 사용자 지정 컴포넌트는 플래그의 영향을 받지 않으며 폼 제출 시 기본값으로 재설정되지 않습니다.
- `enter_to_submit`(bool, Keyword only) : 사용자가 폼 내부의 위젯과 상호 작용하는 동안 Enter 키를 누를 때 폼을 제출할지 여부입니다.
  - True(기본값) : 폼 위젯과 상호 작용하는 동안 Enter 키를 누르면 폼의 첫 번째 `st.form_submit_button`을 클릭하는 것과 동일합니다.
  - False : 사용자가 폼을 제출하려면 `st.form_submit_button`을 클릭해야 합니다. 폼의 첫 번째 `st.form_submit_button`이 비활성화되면 폼은 enter_to_submit=False로 제출 동작을 재정의합니다.
- `border`(bool, Keyword only) : 폼 주위에 테두리를 표시할지 여부입니다. 기본값은 True입니다.
  - 테두리를 표시하지 않으면 폼의 위젯과 상호 작용해도 아무 효과가 없으므로 보는 사람에게 혼란을 줄 수 있습니다. 다른 테두리가 있거나 `st.expender` 때문에 양식이 작은 경우, 예를 들면 텍스트 입력과 제출 버튼만 있는 경우에만 테두리를 제거해야 합니다.

<br>

**표기법**

`st.form` 컨테이너에 요소를 추가하려면 with 표기법을 사용하거나 개체에서 직접 메서드를 호출하면 됩니다.

- `st.form` 객체 표기법
  ```python
  st.form(key).widget_name
  ```
- with 표기법
  ```python
  with st.form(key):
      st.widget_name
  ```

<br>

**예시 1**

다음 예시는 with 표기법을 사용하여 `st.form`에 다양한 요소를 삽입할 수 있습니다.

```python
import streamlit as st

with st.form("my_form"):
    st.write("Inside the form")
    slider_val = st.slider("Form slider")
    checkbox_val = st.checkbox("Form checkbox")

    # Every form must have a submit button.
    submitted = st.form_submit_button("Submit")
    if submitted:
        st.write("slider", slider_val, "checkbox", checkbox_val)
st.write("Outside the form")
```

# st.popover

`st.popover`는 다중 요소 컨테이너를 팝오버로 삽입합니다. 버튼과 같은 요소와 버튼을 클릭하면 열리는 컨테이너로 구성됩니다. 열린 팝오버 내부의 위젯과 상호 작용하면 팝오버를 열어둔 상태에서 앱이 다시 실행됩니다. 팝오버 외부를 클릭하면 닫힙니다. 이때, 팝오버를 열고 닫아도 재실행되지 않으며, 다른 팝오버 안에 팝오버를 넣을 수 없습니다.

```python
st.popover(label, *, help=None, icon=None, disabled=False, use_container_width=False)
```

<br>

**매개변수**

- `label`(str) : 팝오버 컨테이너를 여는 버튼의 레이블입니다. 일부 마크다운 문법을 지원합니다. 이미지는 아이콘처럼 표시되며 최대 높이는 글꼴 높이와 같습니다. 지원되지 않는 마크다운 요소는 래핑이 해제되어 해당 하위 요소(텍스트 콘텐츠)만 렌더링됩니다. 지원되지 않는 요소는 백슬래시 이스케이프를 사용하여 리터럴 문자로 표시합니다. ex) 1\. 정렬된 목록이 아님
- `help`(str, None, Keyword only) : 사용자에게 팝오버 컨테이너의 도움말 텍스트를 제공하기 위해 사용합니다. 지정한 도움말 텍스트는 팝오버 버튼 위로 마우스를 가져가면 툴팁으로 설명이 표시됩니다. 설명에는 마크다운을 사용하실 수 있습니다. None(기본값)이면 설명이 표시되지 않습니다.
- `icon`(str, None, Keyword only) : 버튼 레이블 옆에 표시할 이모티콘 또는 아이콘입니다.
  - None(기본값) : 아이콘이 표시되지 않습니다.
  - 한 글자 이모티콘을 설정할 수 있으며, 이모티콘 단축 코드는 지원되지 않습니다.
    ex) icon="🚨", icon="🔥"
  - 소재 기호 라이브러리의 아이콘(둥근 스타일)으로 ":소재/아이콘*이름:" 형식의 아이콘(여기서 "아이콘*이름"은 snake 표기로 된 아이콘 이름).
    ex) icon=":material/thumb_up:"은 👍🏻 아이콘을 표시합니다.
    https://fonts.google.com/icons?icon.set=Material+Symbols&icon.style=Rounded 글꼴 라이브러리에서 추가 아이콘을 찾을 수 있습니다.
- `disabled`(bool, Keyword only) : 팝오버 버튼을 활성화하거나 비활성화할 수 있습니다.
  - False(기본값) : 팝오버 버튼 활성화
  - True : 팝오버 버튼 비활성화, 비활성화된 위젯은 사용자가 어떠한 조작을 입력할 수 없습니다.
- `use_container_width`(bool, Keyword only) : 버튼의 너비를 부모(상위) 컨테이너의 너비로 설정할지 여부를 지정합니다. 만약, 버튼의 콘텐츠가 상위 컨테이너보다 넓으면 콘텐츠가 줄 바꿈됩니다. 팝오버 컨테이너의 최소 너비는 해당 버튼의 너비와 일치합니다. 팝오버 컨테이너는 컨테이너의 콘텐츠에 맞게 버튼보다 더 넓을 수 있습니다.
  - True : 버튼의 너비를 부모(상위) 컨테이너의 너비와 일치하도록 설정합니다.
  - False(기본값) : 기본 버튼의 너비, 콘텐츠에 맞게 버튼의 너비를 설정합니다.
- `use_container_width`(bool, Keyword only) : 버튼의 너비를 확장하여 상위 컨테이너를 채울지 여부입니다. 두 경우 모두 버튼의 콘텐츠가 부모 컨테이너보다 넓으면 콘텐츠가 줄 바꿈됩니다.
  - False(기본값) : 버튼의 크기를 콘텐츠에 맞게 조정합니다.
  - True : 버튼의 너비가 해당 부모 컨테이너와 일치합니다.

<br>

**표기법**

`st.popover` 컨테이너에 요소를 추가하려면 with 표기법을 사용하거나 개체에서 직접 메서드를 호출하면 됩니다.

- `st.popover` 객체 표기법
  ```python
  st.popover(label).widget_name
  ```
- with 표기법
  ```python
  with st.popover(label):
      st.widget_name
  ```

<br>

**예시 1**

다음 예시는 with 표기법을 사용하여 `st.popover`에 다양한 요소를 삽입할 수 있습니다.

```python
import streamlit as st

with st.popover("Open popover"):
    st.markdown("Hello World 👋")
    name = st.text_input("What's your name?")

st.write("Your name:", name)
```

# st.sidebar

`st.sidebar`는 사이드바 컨테이너를 생성하여 다양한 요소들을 사이드바에 배치할 수 있습니다. 사이드바는 화면 왼쪽에 고정되어 있어 사용자가 주요 콘텐츠에 집중할 수 있으며, 위젯을 사이드바에 배치함으로써 앱에 다양한 상호작용을 구성할 수 있습니다. 또한, 사용자는 사이드바의 오른쪽 테두리를 드래그 앤 드롭하여 크기를 자유롭게 조정할 수 있습니다.

<br>

**표기법**

`st.sidebar` 객체 표기법과 with 표기법을 사용하여 요소를 전달할 수 있습니다.

- `st.sidebar` 객체 표기법

  ```python
  st.sidebar.[element_name]
  ```

  ```python
  st.sidebar.widget_name
  ```

  :::div{.callout}

  `st.echo`, `st.spinner`, `st.toast`은 객체 표기법을 지원하지 않습니다. 이 경우에는 with 표기법을 사용해주시길 바랍니다.

  :::

- with 표기법
  ```python
  with st.sidebar:
      st.widget_name
  ```

<br>

**예시 1**

다음 예시는 사이드 바에 `st.selectbox`와 `st.checkbox` 위젯과 `st.echo`, `st.spinner`를 추가한 예시입니다.

```sql
import streamlit as st
import time

add_selectbox = st.sidebar.selectbox("어떤 차트를 조회할까요?",
                                     ("막대", "꺾은선", "히스토그램", "히트맵"))

with st.sidebar:
    st.checkbox("제목")
    st.checkbox("축제목")
    st.checkbox("눈금선")
    st.checkbox("범례")

with st.sidebar:
    with st.echo():
        st.write("코드 블록 입니다.")

    with st.spinner("Loading..."):
        time.sleep(5)
    st.success("끝!")
```

# st.tabs

`st.tabs`는 탭으로 구분된 여러 페이지를 생성할 수 있는 컨테이너를 제공하여, 한 페이지 내에서 다양한 콘텐츠를 구분해서 보여줄 수 있습니다. 사용자는 원하는 탭을 클릭해 관련된 내용을 손쉽게 탐색할 수 있으며, 이는 콘텐츠 그룹 간의 이동을 보다 직관적으로 만들어주는 유용한 탐색 요소입니다.

```python
st.tabs(tabs)
```

:::div{.callout}

모든 탭의 모든 콘텐츠는 항상 프론트엔드로 전송되어 렌더링됩니다. 조건부 렌더링은 현재 지원되지 않습니다.

:::

<br>

**매개변수**

- `tabs`(list of str) : 리스트에 포함된 각 문자열마다 탭을 생성합니다. 첫 번째 탭은 기본적으로 선택됩니다. 문자열은 탭의 이름으로 사용되며, 선택적으로 Markdown을 포함할 수 있습니다. Markdown을 사용하여 다음과 같은 요소를 지원합니다. 지원되지 않는 마크다운 요소는 래핑이 해제되어 해당 하위 요소(텍스트 콘텐츠)만 렌더링됩니다. 지원되지 않는 요소는 백슬래시 이스케이프를 사용하여 리터럴 문자로 표시합니다. ex) 1\. 정렬된 목록이 아님

<br>

**반환값**

- list of containers : 컨테이너 객체 리스트입니다.

<br>

**표기법**

`st.tabs` 객체 표기법과 with 표기법을 사용하여 요소를 전달할 수 있습니다.

- `st.tabs` 객체 표기법
  ```python
  st.tabs.widget_name
  ```
- with 표기법
  ```python
  with st.tabs:
      st.widget_name
  ```

<br>

**예시 1**

다음 예시는 반환된 객체에서 직접 메서드를 호출하여 `st.tabs`에 다양한 요소를 삽입할 수 있습니다.

```python
import streamlit as st

tab1, tab2 = st.tabs(["탭 이름 1", "탭 이름 2"])

tab1.write("이것은 첫 번째 탭입니다.")

tab2.write("이것은 두 번째 탭입니다.")
```
