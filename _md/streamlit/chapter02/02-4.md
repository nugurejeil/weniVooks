인터렉티브 위젯(Input widgets)은 사용자와의 상호작용이 가능한 대화형 요소로, 사용자가 직접 값을 입력하거나 선택할 수 있습니다. 버튼, 슬라이더 등 다양한 형태로 제공되며, 사용자 친화적인 인터페이스를 제공합니다.

이러한 위젯을 애플리케이션에 통합하면, 사용자의 입력에 따라 애플리케이션의 동작을 실시간으로 변화시킬 수 있어 동적이고 반응형인 앱을 구현할 수 있습니다.

# **버튼 요소**

## st.button

`st.button`은 사용자가 클릭할 수 있는 버튼을 생성합니다. 주로 특정 동작을 실행하는데 사용됩니다.

```python
st.button(label, key=None, help=None, on_click=None, args=None, kwargs=None, *,
          type="secondary", icon=None, disabled=False, use_container_width=False)
```

**매개변수**

- `label`(str) : 사용자에게 버튼이 어떤 용도로 사용되는지 간략히 설명하는 레이블(Label)을 지정합니다. 버튼 최대 높이는 글꼴 높이와 같습니다. 일부 마크다운 문법을 지원하며, 지원되지 않는 마크다운 요소는 래핑이 해제되어 하위 요소(텍스트 콘텐츠)만 렌더링됩니다. 지원되지 않는 요소는 백슬래시 이스케이프를 사용하여 리터럴 문자로 표시합니다. ex) 1\. 정렬된 목록이 아님
- `key`(str, int) : 위젯의 고유 키로 사용할 문자열 또는 정수로 요소에 ID를 부여하는데 사용합니다. `key`가 생략된 경우, 위젯의 내용을 기반으로 키가 생성됩니다. `key`는 위젯의 상태를 식별하고 추적하기 때문에 동일한 유형의 여러 위젯은 동일한 키를 공유할 수 없습니다. 만약, 동일 위젯이 동일 `label`을 가지고 버튼을 생성하였을 때, 동일한 키를 공유할 수 없어 DuplicateWidgetID 에러가 발생합니다.
- `help`(str, None) : 사용자에게 버튼의 도움말 텍스트를 제공하기 위해 사용합니다. 지정한 도움말 텍스트는 버튼 옆에 "?" 아이콘으로 표시되며, 버튼 위로 마우스를 가져가면 툴팁으로 설명이 표시됩니다. 설명에는 마크다운을 사용하실 수 있습니다. None(기본값)이면 설명이 표시되지 않습니다.
- `on_click`(callable) : 버튼을 클릭할 때 호출되는 선택적 콜백 함수입니다. 콜백 함수 선언 시, 파라미터명은 `args`, `kwargs` 대신 원하는 명칭을 지정할 수 있습니다. 단, `args`는 \*(변수명), `kwargs`는 \*\*(변수명) 형태로 지정해야 합니다.
- `args`(tuple) : 콜백에 전달할 선택적 인수 튜플입니다.
- `kwargs`(dict) : 콜백에 전달할 선택적 `kwargs` 딕셔너리입니다.
- `type`(Keyword only) : 버튼 유형을 지정합니다.
  - secondary(기본값) : 일반 버튼, 버튼의 배경이 앱의 배경색과 조화를 이루어 일반적으로 강조됩니다.
  - primary : 추가 강조가 있는 버튼, 버튼의 배경을 앱의 기본 색상으로 설정하여 강조합니다.
  - tertiary : 버튼이 테두리나 배경이 없는 일반 텍스트로 은은하게 강조됩니다.
- `icon`(str, None, Keyword only) : 버튼 레이블 옆에 표시할 이모티콘 또는 아이콘입니다.
  - None(기본값) : 아이콘이 표시되지 않습니다.
  - 한 글자 이모티콘을 설정할 수 있으며, 이모티콘 단축 코드는 지원되지 않습니다.
    ex) icon="🚨", icon="🔥"
  - 소재 기호 라이브러리의 아이콘(둥근 스타일)으로 ":소재/아이콘*이름:" 형식의 아이콘(여기서 "아이콘*이름"은 snake 표기로 된 아이콘 이름).
    ex) icon=":material/thumb_up:"은 👍🏻 아이콘을 표시합니다.
    https://fonts.google.com/icons?icon.set=Material+Symbols&icon.style=Rounded 글꼴 라이브러리에서 추가 아이콘을 찾을 수 있습니다.
- `disabled`(bool, Keyword only) : 버튼을 활성화하거나 비활성화할 수 있습니다.
  - False(기본값) : 버튼 활성화
  - True : 버튼 비활성화, 비활성화된 위젯은 사용자가 어떠한 조작을 입력할 수 없습니다.
- `use_container_width`(bool, Keyword only) : 버튼의 너비를 부모(상위) 컨테이너의 너비로 설정할지 여부를 지정합니다. 만약, 버튼의 콘텐츠가 상위 컨테이너보다 넓으면 콘텐츠가 줄 바꿈됩니다.
  - True : 버튼의 너비를 부모(상위) 컨테이너의 너비와 일치하도록 설정합니다.
  - False(기본값) : 기본 버튼의 너비, 콘텐츠에 맞게 버튼의 너비를 설정합니다.

**반환값**

- bool : 앱을 마지막으로 실행할 때 버튼을 클릭한 경우 True, 그렇지 않으면 False입니다.

**예시 1**

다음 예시는 사용자가 버튼을 클릭했을때 True 값을 반환합니다. 함수는 실제 상태를 반환하므로 IF문을 수행하여 버튼의 반환 상태를 표시합니다.

```python
import streamlit as st

if st.button("Say hello"):
    st.write("Why hello there")
else:
    st.write("Goodbye")
```

**예시 5**

다음 예시는 버튼을 클릭했을때 콜백 함수를 호출합니다.

```python
import streamlit as st

def handle_on_click():
    st.balloons()
    print("clicked on_click button")

def handle_on_click_args(*args):
    st.snow()
    print(f"clicked on_click args button with args={args}")

def handle_on_click_kwargs(**kwargs):
    st.snow()
    print(f"clicked on_click kwargs button with kwargs={kwargs}")

st.button("on_click", on_click=handle_on_click)
st.button("on_click args", on_click=handle_on_click_args, args=("123",))
st.button("on_click kwargs", on_click=handle_on_click_kwargs, kwargs={"one":1})
```

`on_click` 버튼을 클릭하면 handle_on_click 함수가 실행됩니다.

## st.download_button

`st.download_button`은 사용자가 앱에서 직접 파일을 다운로드하려는 경우 다운로드 버튼을 생성합니다. 다운로드할 데이터는 사용자가 연결되어 있는 동안 데이터는 메모리에 저장되므로 메모리를 보존하기 위해 파일 크기를 수백 메가바이트 이하로 유지하는 것이 좋습니다. 사용자가 다운로드 버튼을 클릭할 때, 앱이 재실행되지 않도록 하려면 다운로드 버튼을 [fragment](https://docs.streamlit.io/develop/concepts/architecture/fragments)으로 감싸는 것이 좋습니다.

```python
st.download_button(label, data, file_name=None, mime=None, key=None, help=None,
                   on_click="rerun", args=None, kwargs=None, *, type="secondary",
                   icon=None, disabled=False, use_container_width=False)
```

**매개변수**

- `label`(str) : 사용자에게 버튼이 어떤 용도로 사용되는지 간략히 설명하는 레이블(Label)을 지정합니다. `label`에는 마크다운을 포함할 수 있습니다. 이미지는 아이콘처럼 표시되며 최대 높이는 글꼴 높이와 같습니다. 지원되지 않는 마크다운 요소는 래핑이 해제되어 해당 하위 요소(텍스트 콘텐츠)만 렌더링됩니다. 지원되지 않는 요소는 백슬래시 이스케이프를 사용하여 리터럴 문자로 표시합니다. ex) 1\. 정렬된 목록이 아님
- `data`(str, bytes, file) : 다운로드할 데이터로 파일의 내용입니다. 불필요한 재계산을 방지하려면 다운로드할 데이터를 변환할 때 캐싱을 사용하시길 바랍니다.
- `file_name`(str) : 다운로드할 파일명입니다. 파일명을 지정하지 않으면 이름이 자동으로 생성됩니다.
- `mime`(str, None) : 데이터의 MIME 유형입니다.

  - None(기본값) : 데이터 값에 따라 MIME 유형을 설정합니다.
  - text/plain : 문자열 또는 텍스트 파일입니다. ex) str 또는 io.TextIOWrapper 객체
  - application/octet-stream : 데이터가 바이너리 파일 또는 바이트 ex) 바이트, io.BytesIO, io.BufferedReader 또는 io.RawIOBase 객체
    <aside>
    💡 **MIME(Multipurpose Internet Mail Extensions) 유형**

    MIME 유형은 인터넷에서 데이터를 표현하고 전송하는 데 사용되는 데이터 형식을 식별하기 위한 표준화된 방법입니다.
    주로 웹 브라우저가 서버로부터 받은 파일의 유형을 인식하고 적절한 방식으로 처리하고 데이터를 표시, 다운로드할 수 있도록 돕는 데 사용됩니다.

    MIME 유형은 일반적으로 파일의 확장자로 식별되며, 각각의 유형은 고유한 식별자로 구성됩니다.

    [MIME 타입의 전체 목록 - HTTP | MDN](https://developer.mozilla.org/ko/docs/Web/HTTP/Basics_of_HTTP/MIME_types/Common_types)

    </aside>

- `key`(str, int) : 위젯의 고유 키로 사용할 문자열 또는 정수로 요소에 ID를 부여하는데 사용합니다. `key`가 생략된 경우, 위젯의 내용을 기반으로 키가 생성됩니다. `key`는 위젯의 상태를 식별하고 추적하기 때문에 동일한 유형의 여러 위젯은 동일한 키를 공유할 수 없습니다. 만약, 동일 위젯이 동일 `label`을 가지고 버튼을 생성하였을 때, 동일한 키를 공유할 수 없어 DuplicateWidgetID 에러가 발생합니다.
- `help`(str, None) : 사용자에게 버튼의 도움말 텍스트를 제공하기 위해 사용합니다. 지정한 도움말 텍스트는 버튼 옆에 "?" 아이콘으로 표시되며, 버튼 위로 마우스를 가져가면 툴팁으로 설명이 표시됩니다. 설명에는 마크다운을 사용하실 수 있습니다. None(기본값)이면 설명이 표시되지 않습니다.
- `on_click`(callable, "rerun", "ignore", None) : 버튼이 사용자 상호작용에 반응하는 방식으로 버튼을 클릭할 때 호출되는 선택적 콜백 함수입니다. 버튼이 재실행을 트리거할지 여부와 콜백 함수가 호출되는지 여부를 제어합니다. 콜백 함수 선언 시, 파라미터명은 `args`, `kwargs` 대신 원하는 명칭을 지정할 수 있습니다. 단, `args`는 \*(변수명), `kwargs`는 \*\*(변수명) 형태로 지정해야 합니다.
  - rerun(기본값) : 사용자가 파일을 다운로드하고 앱이 다시 실행됩니다. 콜백 함수가 호출되지 않습니다.
  - ignore : 사용자가 파일을 다운로드하고 앱이 재실행되지 않습니다. 콜백 함수가 호출되지 않습니다.
  - callable : 사용자가 파일을 다운로드하고 앱이 다시 실행됩니다. callable은 나머지 앱보다 먼저 호출됩니다.
  - None : "rerun"과 동일하게 동작하며, None은 이전 버전과의 호환성을 위해 존재하며 사용해서는 안 됩니다.
- `args`(tuple) : 콜백에 전달할 선택적 인수 튜플입니다.
- `kwargs`(dict) : 콜백에 전달할 선택적 `kwargs` 딕셔너리입니다.
- `type`("primary", "secondary", "tertiary", Keyword only) : 버튼 유형을 지정합니다.
  - secondary(기본값) : 일반 버튼, 버튼의 배경이 앱의 배경색과 조화를 이루어 일반적으로 강조됩니다.
  - primary : 추가 강조가 있는 버튼, 버튼의 배경을 앱의 기본 색상으로 설정하여 강조합니다.
  - tertiary : 버튼이 테두리나 배경이 없는 일반 텍스트로 은은하게 강조됩니다.
- `icon`(str, None, Keyword only) : 버튼 레이블 옆에 표시할 이모티콘 또는 아이콘입니다.
  - None(기본값) : 아이콘이 표시되지 않습니다.
  - 한 글자 이모티콘을 설정할 수 있으며, 이모티콘 단축 코드는 지원되지 않습니다.
    ex) icon="🚨", icon="🔥"
  - 소재 기호 라이브러리의 아이콘(둥근 스타일)으로 ":소재/아이콘*이름:" 형식의 아이콘(여기서 "아이콘*이름"은 snake 표기로 된 아이콘 이름).
    ex) icon=":material/thumb_up:"은 👍🏻 아이콘을 표시합니다.
    https://fonts.google.com/icons?icon.set=Material+Symbols&icon.style=Rounded 글꼴 라이브러리에서 추가 아이콘을 찾을 수 있습니다.
- `disabled`(bool, Keyword only) : 다운로드 버튼을 활성화하거나 비활성화할 수 있습니다.
  - False(기본값) : 다운로드 버튼 활성화
  - True : 다운로드 버튼 비활성화, 비활성화된 위젯은 사용자가 어떠한 조작을 입력할 수 없습니다.
- `use_container_width`(bool, Keyword only) : 버튼의 너비를 부모(상위) 컨테이너의 너비로 설정할지 여부를 지정합니다. 만약, 버튼의 콘텐츠가 상위 컨테이너보다 넓으면 콘텐츠가 줄 바꿈됩니다.
  - True : 버튼의 너비를 부모(상위) 컨테이너의 너비와 일치하도록 설정합니다.
  - False(기본값) : 기본 버튼의 너비, 콘텐츠에 맞게 버튼의 너비를 설정합니다.

**반환값**

- bool : 앱을 마지막으로 실행할 때 버튼을 클릭한 경우 True, 그렇지 않으면 False입니다.

```python
import streamlit as st
```

```python
if st.button("Prepare download"): # 다운로드 준비 버튼
    # 다운로드 버튼 - 재실행하지 않도록 설정함
    st.download_button(
        label="Download text",
        data=message,
        file_name="message.txt",
        on_click="ignore",  # 중요! 다운로드 버튼 클릭 시 재실행 방지
        type="primary",
        icon=":material/download:",
    )
```

## **st.form_submit_button**

`st.form_submit_button`은 form 제출 버튼을 생성합니다. 이 버튼을 클릭하면 form 내의 모든 위젯 값이 사용자의 브라우저에서 Streamlit 서버로 일괄 전송됩니다. 모든 form에는 하나 이상의 `st.form_submit_button`이 있어야하며, `st.form_submit_button`은 form 외부에 존재할 수 없습니다.

```python
st.form_submit_button(label="Submit", help=None, on_click=None, args=None, kwargs=None, *, type="secondary", icon=None, disabled=False, use_container_width=False)
```

**매개변수**

- `label`(str) : 사용자에게 버튼이 어떤 용도로 사용되는지 간략히 설명하는 레이블(Label)을 지정합니다. `label`에는 마크다운을 포함할 수 있습니다. 이미지는 아이콘처럼 표시되며 최대 높이는 글꼴 높이와 같습니다. 지원되지 않는 마크다운 요소는 래핑이 해제되어 해당 하위 요소(텍스트 콘텐츠)만 렌더링됩니다. 지원되지 않는 요소는 백슬래시 이스케이프를 사용하여 리터럴 문자로 표시합니다. ex) 1\. 정렬된 목록이 아님
- `help`(str, None) : 사용자에게 버튼의 도움말 텍스트를 제공하기 위해 사용합니다. 지정한 도움말 텍스트는 버튼 옆에 "?" 아이콘으로 표시되며, 버튼 위로 마우스를 가져가면 툴팁으로 설명이 표시됩니다. 설명에는 마크다운을 사용하실 수 있습니다. None(기본값)이면 설명이 표시되지 않습니다.
- `on_click`(callable) : 버튼을 클릭할 때 호출되는 선택적 콜백 함수입니다.
- `args`(tuple) : 콜백에 전달할 선택적 인수 튜플입니다.
- `kwargs`(dict) : 콜백에 전달할 선택적 `kwargs` 딕셔너리입니다.
- `type`("primary", "secondary", "tertiary", Keyword only) : 버튼 유형을 지정합니다.
  - secondary(기본값) : 일반 버튼, 버튼의 배경이 앱의 배경색과 조화를 이루어 일반적으로 강조됩니다.
  - primary : 추가 강조가 있는 버튼, 버튼의 배경을 앱의 기본 색상으로 설정하여 강조합니다.
  - tertiary : 버튼이 테두리나 배경이 없는 일반 텍스트로 은은하게 강조됩니다.
- `icon`(str, None, Keyword only) : 버튼 레이블 옆에 표시할 이모티콘 또는 아이콘입니다.
  - None(기본값) : 아이콘이 표시되지 않습니다.
  - 한 글자 이모티콘을 설정할 수 있으며, 이모티콘 단축 코드는 지원되지 않습니다.
    ex) icon="🚨", icon="🔥"
  - 소재 기호 라이브러리의 아이콘(둥근 스타일)으로 ":소재/아이콘*이름:" 형식의 아이콘(여기서 "아이콘*이름"은 snake 표기로 된 아이콘 이름).
    ex) icon=":material/thumb_up:"은 👍🏻 아이콘을 표시합니다.
    https://fonts.google.com/icons?icon.set=Material+Symbols&icon.style=Rounded 글꼴 라이브러리에서 추가 아이콘을 찾을 수 있습니다.
- `disabled`(bool, Keyword only) : 버튼을 활성화하거나 비활성화할 수 있습니다. 양식의 첫 번째 `st.form_submit_button`이 비활성화되면 양식은 enter_to_submit=False로 제출 동작을 재정의합니다.
  - False(기본값) : 버튼 활성화, 버튼과 상호 작용할 수 있습니다.
  - True : 버튼 비활성화, 비활성화된 위젯은 회색으로 표시되어 사용자가 어떠한 조작을 입력할 수 없습니다.
- `use_container_width`(bool, Keyword only) : 버튼의 너비를 부모(상위) 컨테이너의 너비로 설정할지 여부를 지정합니다. 만약, 버튼의 콘텐츠가 상위 컨테이너보다 넓으면 콘텐츠가 줄 바꿈됩니다.
  - True : 버튼의 너비를 부모(상위) 컨테이너의 너비와 일치하도록 설정합니다.
  - False(기본값) : 기본 버튼의 너비, 콘텐츠에 맞게 버튼의 너비를 설정합니다.

**반환값**

- bool : 버튼을 클릭한 경우 True입니다.

## st.link_button

`st.link_button`는 링크 버튼 요소를 표시합니다. 클릭하면 지정된 URL로 연결되는 새 탭이 열립니다. 만약, 버튼을 눌러 앱 내부의 다른 경로로 이동하면, 기존 세션은 종료되고 새로운 세션이 생성되어 페이지가 로드됩니다.

```python
st.link_button(label, url, *, help=None, type="secondary", icon=None, disabled=False, use_container_width=False)
```

**매개변수**

- `label`(str) : 사용자에게 버튼이 어떤 용도로 사용되는지 간략히 설명하는 레이블(Label)을 지정합니다. 기본값은 Submit입니다. 레이블에는 마크다운을 포함할 수 있습니다. 이미지는 아이콘처럼 표시되며 최대 높이는 글꼴 높이와 같습니다. 지원되지 않는 마크다운 요소는 래핑이 해제되어 해당 하위 요소(텍스트 콘텐츠)만 렌더링됩니다. 지원되지 않는 요소는 백슬래시 이스케이프를 사용하여 리터럴 문자로 표시합니다. ex) 1\. 정렬된 목록이 아님
- `url`(str) : 사용자 클릭 시 열릴 URL입니다.
- `help`(str, None, Keyword only) : 사용자에게 버튼의 도움말 텍스트를 제공하기 위해 사용합니다. 지정한 도움말 텍스트는 버튼 옆에 "?" 아이콘으로 표시되며, 버튼 위로 마우스를 가져가면 툴팁으로 설명이 표시됩니다. 설명에는 마크다운을 사용하실 수 있습니다. None(기본값)이면 설명이 표시되지 않습니다.
- `type`("primary", "secondary", "tertiary", Keyword only) : 버튼 유형을 지정합니다.
  - secondary(기본값) : 일반 버튼, 버튼의 배경이 앱의 배경색과 조화를 이루어 일반적으로 강조됩니다.
  - primary : 추가 강조가 있는 버튼, 버튼의 배경을 앱의 기본 색상으로 설정하여 강조합니다.
  - tertiary : 버튼이 테두리나 배경이 없는 일반 텍스트로 은은하게 강조됩니다.
- `icon`(str, None, Keyword only) : 버튼 레이블 옆에 표시할 이모티콘 또는 아이콘입니다.
  - None(기본값) : 아이콘이 표시되지 않습니다.
  - 한 글자 이모티콘을 설정할 수 있으며, 이모티콘 단축 코드는 지원되지 않습니다.
    ex) icon="🚨", icon="🔥"
  - 소재 기호 라이브러리의 아이콘(둥근 스타일)으로 ":소재/아이콘*이름:" 형식의 아이콘(여기서 "아이콘*이름"은 snake 표기로 된 아이콘 이름).
    ex) icon=":material/thumb_up:"은 👍🏻 아이콘을 표시합니다.
    https://fonts.google.com/icons?icon.set=Material+Symbols&icon.style=Rounded 글꼴 라이브러리에서 추가 아이콘을 찾을 수 있습니다.
- `disabled`(bool, Keyword only) : 버튼을 활성화하거나 비활성화할 수 있습니다. 양식의 첫 번째 `st.form_submit_button`이 비활성화되면 양식은 enter_to_submit=False로 제출 동작을 재정의합니다.
  - False(기본값) : 버튼 활성화, 버튼과 상호 작용할 수 있습니다.
  - True : 버튼 비활성화, 비활성화된 위젯은 회색으로 표시되어 사용자가 어떠한 조작을 입력할 수 없습니다.
- `use_container_width`(bool, Keyword only) : 버튼의 너비를 부모(상위) 컨테이너의 너비로 설정할지 여부를 지정합니다. 만약, 버튼의 콘텐츠가 상위 컨테이너보다 넓으면 콘텐츠가 줄 바꿈됩니다.
  - True : 버튼의 너비를 부모(상위) 컨테이너의 너비와 일치하도록 설정합니다.
  - False(기본값) : 기본 버튼의 너비, 콘텐츠에 맞게 버튼의 너비를 설정합니다.

**예시 1**

다음 예시는 버튼을 누르면 해당 링크로 이동하게 해줍니다.

```python
import streamlit as st

st.link_button("Go to weniv", "https://weniv.co.kr")
```

## st.page_link

`st.page_link`는 다중 페이지 앱의 다른 페이지 또는 외부 페이지로 연결되는 링크를 표시합니다.

- 다중 페이지 앱의 다른 페이지가 지정된 경우 : `st.page_link`를 클릭하면 현재 페이지 실행이 중지되고 사이드바 탐색에서 해당 페이지를 클릭한 것처럼 지정된 페이지가 실행됩니다.
- 외부 페이지가 지정된 경우 : `st.page_link`를 클릭하면 지정된 페이지로 새 탭이 열립니다. 현재 스크립트 실행은 완료되지 않은 경우 계속됩니다.

```python
st.page_link(page, *, label=None, icon=None, help=None, disabled=False, use_container_width=None)
```

**매개변수**

- `page`(str, Path, st.Page) : 파일 경로(기본 스크립트 기준) 또는 전환할 페이지를 나타내는 `st.Page` 또는 외부 페이지의 URL("http://" 또는 "https://")입니다.
- `label`(str, Keyword only) : 사용자에게 페이지 링크가 어떤 용도로 사용되는지 간략히 설명하는 레이블(Label)을 지정합니다. 외부 페이지에는 레이블이 필요합니다. 기본값은 Submit입니다. 레이블에는 마크다운을 포함할 수 있습니다. 이미지는 아이콘처럼 표시되며 최대 높이는 글꼴 높이와 같습니다. 지원되지 않는 마크다운 요소는 래핑이 해제되어 해당 하위 요소(텍스트 콘텐츠)만 렌더링됩니다. 지원되지 않는 요소는 백슬래시 이스케이프를 사용하여 리터럴 문자로 표시합니다. ex) 1\. 정렬된 목록이 아님
- `icon`(str, None, Keyword only) : 버튼 레이블 옆에 표시할 이모티콘 또는 아이콘입니다.
  - None(기본값) : 아이콘이 표시되지 않습니다.
  - 한 글자 이모티콘을 설정할 수 있으며, 이모티콘 단축 코드는 지원되지 않습니다.
    ex) icon="🚨", icon="🔥"
  - 소재 기호 라이브러리의 아이콘(둥근 스타일)으로 ":소재/아이콘*이름:" 형식의 아이콘(여기서 "아이콘*이름"은 snake 표기로 된 아이콘 이름).
    ex) icon=":material/thumb_up:"은 👍🏻 아이콘을 표시합니다.
    https://fonts.google.com/icons?icon.set=Material+Symbols&icon.style=Rounded 글꼴 라이브러리에서 추가 아이콘을 찾을 수 있습니다.
- `help`(str, None, Keyword only) : 사용자에게 링크의 도움말 텍스트를 제공하기 위해 사용합니다. 지정한 도움말 텍스트는 링크 옆에 "?" 아이콘으로 표시되며, 링크를 마우스를 가리키면 툴팁으로 설명이 표시됩니다. 설명에는 마크다운을 사용하실 수 있습니다. None(기본값)이면 설명이 표시되지 않습니다.
- `disabled`(bool, Keyword only) : 페이지 링크를 활성화하거나 비활성화할 수 있습니다.
  - False(기본값) : 페이지 링크 활성화
  - True : 페이지 링크 비활성화
- `use_container_width`(bool, Keyword only) : 링크의 너비를 부모(상위) 컨테이너의 너비로 설정할지 여부를 지정합니다. 만약, 링크의 콘텐츠가 상위 컨테이너보다 넓으면 콘텐츠가 줄 바꿈됩니다.
  - True(기본값) : 사이드바에 있는 페이지 링크의 경우
  - False : 기본 앱에 있는 링크의 경우

# **선택 요소(**상호작용을 통한 동적 데이터 시각화**)**

## st.checkbox

`st.checkbox`는 참/거짓 값을 선택할 수 있는 체크박스를 생성합니다. 여러 개의 체크박스를 함께 사용하여 다중 선택이 가능합니다.

```python
st.checkbox(label, value=False, key=None, help=None, on_change=None,
            args=None, kwargs=None, *, disabled=False,
            label_visibility="visible")
```

**매개변수**

- `label`(str) : 사용자에게 체크박스가 어떤 용도로 사용되는지 간략히 설명하는 레이블(Label)을 지정합니다. 일부 마크다운 문법을 지원합니다. 접근성을 위해 빈 레이블을 설정해서는 안 되지만, 필요한 경우 `label_visibility`를 사용하여 숨길 수 있습니다. 향후에는 예외를 발생시켜 빈 레이블을 허용하지 않을 수 있습니다.
- `value`(bool) : 체크박스의 초기 상태를 보여주는 bool 값입니다.
  - True : 체크박스 선택합니다.
  - False(기본값) : 체크박스 선택되지 않습니다.
- `key`(str, int) : 위젯의 고유 키로 사용할 문자열 또는 정수로 요소에 ID를 부여하는데 사용합니다. `key`가 생략된 경우, 위젯의 내용을 기반으로 키가 생성됩니다. `key`는 위젯의 상태를 식별하고 추적하기 때문에 동일한 유형의 여러 위젯은 동일한 키를 공유할 수 없습니다. 만약, 동일 위젯이 동일 `label`을 가지고 버튼을 생성하였을 때, 동일한 키를 공유할 수 없어 DuplicateWidgetID 에러가 발생합니다.
- `help`(str, None) : 사용자에게 체크박스 옆에 도움말 텍스트를 제공하기 위해 사용합니다. 지정한 도움말 텍스트는 체크박스 옆에 "?" 아이콘으로 표시되며, 마우스를 가져다 대면 툴팁으로 표시됩니다. label_visibility="visible"일 때만 툴팁을 표시하며, 설명에는 마크다운을 사용할 수 있습니다. None(기본값)이면 툴팁이 표시되지 않습니다.
- `on_change`(callable) : 체크박스의 값이 변경될 때 호출되는 선택적 콜백 함수입니다.
- `args`(tuple) : 콜백에 전달할 선택적 인수 튜플입니다.
- `kwargs`(dict) : 콜백에 전달할 선택적 `kwargs` 딕셔너리입니다.
- `disabled`(bool, Keyword only) : 체크박스를 활성화하거나 비활성화할 수 있습니다.
  - False(기본값) : 체크박스 활성화
  - True : 체크박스 비활성화
- `label_visibility`("visible", "hidden", "collapsed", Keyword only) : 레이블의 표시 여부입니다.
  - "visible"(기본값) : 레이블을 필드 위에 표시합니다. 필드는 위젯의 값이 입력되는 공간을 의미합니다.
  - "hidden" : 레이블은 표시되지 않지만, 위젯 위에 빈 공간(공백)을 표시하여 위젯을 다른 위젯과 정렬하는 데 도움을 줍니다.(label=""와 동일)
  - "collapsed" : 레이블과 공백이 모두 제거됩니다. 기본값은 `label`을 보여 줍니다.

**반환값**

- bool : 확인란이 선택되어 있는지 여부입니다.

**예시 1**

다음 예시는 체크박스를 체크하면 “체크박스가 선택되었습니다!!”라는 텍스트가 출력됩니다. 체크박스의 `value`값은 기본값이 False이므로 체크박스가 선택되어 있지 않습니다.

```python
import streamlit as st

check = st.checkbox(label="체크박스를 선택하세요!!", value=False)

if check:
    st.write("체크박스가 선택되었습니다!!")
else:
    st.write("체크박스가 선택되지 않았습니다.")
```

다음 예시는 비밀번호 변경 체크박스를 생성하지만, `disabled`를 True로 설정하여 체크박스를 비활성화합니다. `disabled`를 True로 설정하게 되면 체크박스가 비활성화되어 사용자가 해당 체크박스를 선택하거나 수정할 수 없게 됩니다.

```python
password = st.checkbox("비밀번호 변경", disabled = True)

    st.write("비밀번호가 변경되었습니다!")
else:
    st.write("비밀번호를 변경할 수 없습니다!")
```

## st.color_picker

`st.color_picker`는 색상 선택기를 생성합니다. 웹상에서 사용자가 선택한 색상을 그래프, 텍스트, 배경 등의 요소에 적용할 수 있습니다. RGB 값 또는 HEX 코드로 컬러를 선택할 수 있습니다.

```python
st.color_picker(label, value=None, key=None, help=None, on_change=None,
                                args=None, kwargs=None, *, disabled=False,
                                label_visibility="visible")
```

**매개변수**

- `label`(str) : 사용자에게 색상 선택기가 어떤 용도로 사용되는지 간략히 설명하는 레이블입니다. 레이블에는 마크다운을 포함할 수 있습니다. 이미지는 아이콘처럼 표시되며 최대 높이는 글꼴 높이와 같습니다. 지원되지 않는 마크다운 요소는 래핑이 해제되어 해당 하위 요소(텍스트 콘텐츠)만 렌더링됩니다. 지원되지 않는 요소는 백슬래시 이스케이프를 사용하여 리터럴 문자로 표시합니다. ex) 1\. 정렬된 목록이 아님
  접근성을 위해 빈 레이블을 설정해서는 안 되지만, 필요한 경우 `label_visibility`를 사용하여 숨길 수 있습니다. 향후에는 예외를 발생시켜 빈 레이블을 허용하지 않을 수 있습니다.
- `value`(str) : 초기값으로 사용되는 색상 값으로 처음 렌더링될 때의 16진수 값입니다.
  - None(기본값) : 검은색
  - “#00FFAA”, “#000” 형식의 hex 색상 코드를 사용합니다.
- `key`(str, int) : 위젯의 고유 키로 사용할 문자열 또는 정수로 요소에 ID를 부여하는데 사용합니다. `key`가 생략된 경우, 위젯의 내용을 기반으로 키가 생성됩니다. `key`는 위젯의 상태를 식별하고 추적하기 때문에 동일한 유형의 여러 위젯은 동일한 키를 공유할 수 없습니다. 만약, 동일 위젯이 동일 `label`을 가지고 버튼을 생성하였을 때, 동일한 키를 공유할 수 없어 DuplicateWidgetID 에러가 발생합니다.
- `help`(str, None) : 사용자에게 색상 선택기 옆에 도움말 텍스트를 제공하기 위해 사용합니다. 지정한 도움말 텍스트는 색상 선택기 옆에 "?" 아이콘으로 표시되며, 마우스를 가져다 대면 툴팁으로 표시됩니다. label_visibility="visible"일 때만 툴팁을 표시하며, 설명에는 마크다운을 사용할 수 있습니다. None(기본값)이면 툴팁이 표시되지 않습니다.
- `on_change`(callable) : 색상 선택기 값이 변경될 때 호출되는 선택적 콜백 함수입니다.
- `args`(tuple) : 콜백에 전달할 선택적 인수 튜플입니다.
- `kwargs`(dict) : 콜백에 전달할 선택적 `kwargs` 딕셔너리입니다.
- `disabled`(bool, Keyword only) : 색상 선택기를 활성화하거나 비활성화할 수 있습니다.
  - False(기본값) : 색상 선택기 활성화
  - True : 색상 선택기 비활성화, 비활성화된 위젯은 사용자가 어떠한 조작을 입력할 수 없습니다.
- `label_visibility`("visible", "hidden", "collapsed", Keyword only) : 레이블의 표시 여부입니다.
  - "visible"(기본값) : 레이블을 필드 위에 표시합니다. 필드는 위젯의 값이 입력되는 공간을 의미합니다.
  - "hidden" : 레이블은 표시되지 않지만, 위젯 위에 빈 공간(공백)을 표시하여 위젯을 다른 위젯과 정렬하는 데 도움을 줍니다.(label=""와 동일)
  - "collapsed" : 레이블과 공백이 모두 제거됩니다. 기본값은 `label`을 보여 줍니다.

**반환값**

- str : 선택한 색상을 16진수 문자열로 지정합니다.

## st.feedback

`st.feedback`는 피드백 위젯을 표시합니다. 피드백 위젯은 옵션에 설명된 대로 세 가지 스타일로 제공되는 아이콘 기반 버튼 그룹입니다. 일반적으로 채팅 및 AI 앱에서 사용자가 응답을 평가할 수 있도록 하는데 사용됩니다.

```python
st.feedback(options="thumbs", *, key=None, disabled=False, on_change=None, args=None, kwargs=None)
```

**매개변수**

- `options`("thumbs", "faces", "stars") : 사용자에게 표시되는 피드백 옵션 설정 방법입니다.
  - thumbs(기본값) : 엄지손가락 위로 및 아래로 버튼 그룹을 표시합니다.
  - faces : 왼쪽에서 오른쪽으로 증가하는 만족도를 나타내는 얼굴 표정이 있는 5개의 버튼 행을 표시합니다.
  - stars : 별 아이콘 행을 표시하여 사용자가 별 1개부터 5개까지의 등급을 선택할 수 있도록 합니다.
- `key`(str, int, Keyword only) : 위젯의 고유 키로 사용할 문자열 또는 정수로 요소에 ID를 부여하는데 사용합니다. `key`가 생략된 경우, 위젯의 내용을 기반으로 키가 생성됩니다. `key`는 위젯의 상태를 식별하고 추적하기 때문에 동일한 유형의 여러 위젯은 동일한 키를 공유할 수 없습니다. 만약, 동일 위젯이 동일 `label`을 가지고 버튼을 생성하였을 때, 동일한 키를 공유할 수 없어 DuplicateWidgetID 에러가 발생합니다.
- `disabled`(bool, Keyword only) : 색상 선택기를 활성화하거나 비활성화할 수 있습니다.
  - False(기본값) : 색상 선택기를 활성화
  - True : 색상 선택기를 비활성화, 비활성화된 위젯은 사용자가 어떠한 조작을 입력할 수 없습니다.
- `on_change`(callable, Keyword only) : 피드백 위젯의 값이 변경될 때 호출되는 선택적 콜백 함수입니다.
- `args`(tuple, Keyword only) : 콜백에 전달할 선택적 인수 튜플입니다.
- `kwargs`(dict, Keyword only) : 콜백에 전달할 선택적 `kwargs` 딕셔너리입니다.

**반환값**

- int, None : 사용자의 선택을 나타내는 정수로, 0은 가장 낮은 피드백을 나타냅니다. 값이 클수록 긍정적인 피드백이 많음을 나타내며, 선택된 옵션이 없는 경우 위젯은 None을 반환합니다.
  - options="thumbs” : 반환값이 0이면 '좋아요'를, 1이면 '싫어요'
  - options="faces", options="stars” : 반환값은 0(만족도가 가장 낮음)에서 4(만족도가 가장 높음)

**예시 1**

다음 예시는 별표가 있는 피드백 위젯을 표시하고 선택한 별 개수를 표시합니다.

```python
import streamlit as st

sentiment_mapping = ["one", "two", "three", "four", "five"]
selected = st.feedback("stars")
if selected is not None:
    st.markdown(f"You selected {sentiment_mapping[selected]} star(s).")
```

## st.multiselect

`st.multiselect`는 사용자가 여러 개의 항목을 선택할 수 있는 다중 선택 위젯을 생성합니다. 처음에는 아무것도 선택되지 않은 상태로 화면에 출력됩니다.

```python
st.multiselect(label, options, default = None,
               format_func=special_internal_function, key = None, help = None,
               on_change = None, args = None, kwargs = None, *,
               disabled = False, label_visibility = "visible",
               placeholder="Choose an option", max_selections = None)
```

**매개변수**

- `label`(str) : 사용자에게 다중 선택 위젯이 어떤 용도로 사용되는지 간략히 설명하는 레이블입니다. 레이블에는 마크다운을 포함할 수 있습니다. 이미지는 아이콘처럼 표시되며 최대 높이는 글꼴 높이와 같습니다. 지원되지 않는 마크다운 요소는 래핑이 해제되어 해당 하위 요소(텍스트 콘텐츠)만 렌더링됩니다. 지원되지 않는 요소는 백슬래시 이스케이프를 사용하여 리터럴 문자로 표시합니다. ex) 1\. 정렬된 목록이 아님
  접근성을 위해 빈 레이블을 설정해서는 안 되지만, 필요한 경우 `label_visibility`를 사용하여 숨길 수 있습니다. 향후에는 예외를 발생시켜 빈 레이블을 허용하지 않을 수 있습니다.
- `options`(Iterable) : 선택 가능한 항목들이 list, set, Sequence[V], numpy.ndarray, pandas.Series, pandas.DataFrame, pandas.Index 등으로 나타낼 수 있습니다. 사용자가 리스트에서 여러 개의 항목을 선택할 수 있습니다. 옵션이 데이터프레임과 유사한 경우 첫 번째 열이 사용됩니다. 각 레이블은 기본적으로 내부적으로 str으로 형변환됩니다.
- `default`(Iterable of V, V, None) : 기본값들의 리스트입니다. 단일 값일 수도 있습니다.
- `format_func`(function) : 다중 선택 위젯 옵션의 표시를 수정하는 함수입니다. 원시 옵션을 인수로 받고 해당 옵션에 표시할 레이블을 출력해야 합니다. 이 함수는 명령의 반환 값에는 영향을 미치지 않습니다.
- `key`(str, int) : 위젯의 고유 키로 사용할 문자열 또는 정수로 요소에 ID를 부여하는데 사용합니다. `key`가 생략된 경우, 위젯의 내용을 기반으로 키가 생성됩니다. `key`는 위젯의 상태를 식별하고 추적하기 때문에 동일한 유형의 여러 위젯은 동일한 키를 공유할 수 없습니다. 만약, 동일 위젯이 동일 `label`을 가지고 버튼을 생성하였을 때, 동일한 키를 공유할 수 없어 DuplicateWidgetID 에러가 발생합니다.
- `help`(str*,* None) : 사용자에게 다중 선택 위젯 레이블 옆에 도움말 텍스트를 제공하기 위해 사용합니다. 지정한 도움말 텍스트는 다중 선택 위젯 레이블 옆에 "?" 아이콘으로 표시되며, 마우스를 가져다 대면 툴팁으로 표시됩니다. label_visibility="visible"일 때만 툴팁을 표시하며, 설명에는 마크다운을 사용할 수 있습니다. None(기본값)이면 툴팁이 표시되지 않습니다.
- `on_change`(callable) : 다중 선택 위젯 값이 변경될 때 호출되는 선택적 콜백 함수입니다.
- `args`(tuple) : 콜백에 전달할 선택적 인수 튜플입니다.
- `kwargs`(dict) : 콜백에 전달할 선택적 `kwargs` 딕셔너리입니다.
- `max_selections`(int, Keyword only) : 한 번에 선택할 수 있는 최대 선택 항목입니다.
- `placeholder`(str, Keyword only) : 선택된 옵션이 없을 때 표시할 문자열입니다. 기본값은 "옵션 선택"입니다.
- `disabled`(bool, Keyword only) : 다중 선택 위젯을 활성화하거나 비활성화할 수 있습니다.
  - False(기본값) : 다중 선택 위젯 활성화
  - True : 다중 선택 위젯 비활성화, 비활성화된 위젯은 사용자가 어떠한 조작을 입력할 수 없습니다.
- `label_visibility`("visible", "hidden", "collapsed", Keyword only) : 레이블의 표시 여부입니다.
  - "visible"(기본값) : 레이블을 필드 위에 표시합니다. 필드는 위젯의 값이 입력되는 공간을 의미합니다.

  - "collapsed" : 레이블과 공백이 모두 제거됩니다. 기본값은 `label`을 보여 줍니다.

**반환값**

- list : 선택한 옵션이 있는 목록

## st.pills

`st.pills`는 다중 선택 알약 위젯을 표시합니다. 알약 위젯은 옵션이 드롭다운 목록이 아닌 알약 버튼으로 표시되는 `st.selectbox` 또는 `st.multiselect`와 유사합니다.

```python
st.pills(label, options, *, selection_mode="single", default=None, format_func=None, key=None, help=None, on_change=None, args=None, kwargs=None, disabled=False, label_visibility="visible")
```

**매개변수**

- `label`(str) : 사용자에게 알약 위젯이 어떤 용도로 사용되는지 간략히 설명하는 레이블입니다. 레이블에는 마크다운을 포함할 수 있습니다. 이미지는 아이콘처럼 표시되며 최대 높이는 글꼴 높이와 같습니다. 지원되지 않는 마크다운 요소는 래핑이 해제되어 해당 하위 요소(텍스트 콘텐츠)만 렌더링됩니다. 지원되지 않는 요소는 백슬래시 이스케이프를 사용하여 리터럴 문자로 표시합니다. ex) 1\. 정렬된 목록이 아님
  접근성을 위해 빈 레이블을 설정해서는 안 되지만, 필요한 경우 `label_visibility`를 사용하여 숨길 수 있습니다. 향후에는 예외를 발생시켜 빈 레이블을 허용하지 않을 수 있습니다.
- `options`(Iterable of V) : 이터러블의 선택 옵션에 대한 레이블입니다. 선택 가능한 항목들이 list, set, Sequence[V], numpy.ndarray, pandas.Series, pandas.DataFrame, pandas.Index 등으로 나타낼 수 있습니다. 사용자가 리스트에서 여러 개의 항목을 선택할 수 있습니다. 옵션이 데이터프레임과 유사한 경우 첫 번째 열이 사용됩니다. 각 레이블은 기본적으로 내부적으로 str으로 형변환되며, 마크다운을 사용할 수 있습니다.
- `selection_mode`("single", "multi", Keyword only) : 위젯의 선택 모드입니다.
  - "single"(기본값) : 하나의 옵션만 선택할 수 있습니다.
  - "multi" : 여러 옵션을 선택할 수 있습니다.
- `default`(Iterable of V, V, None, Keyword only) : 위젯이 처음 렌더링될 때 위젯의 값입니다.
  - selection_mode="multi" : 리스트, 단일 값 또는 None이 될 수 있습니다.
  - selection_mode="single" : 단일 값 또는 None이 될 수 있습니다.
- `format_func`(function, Keyword only) : 알약 위젯 옵션의 표시를 수정하는 함수입니다. 원시 옵션을 인수로 받고 해당 옵션에 표시할 레이블을 출력해야 합니다. 이 함수는 명령의 반환 값에는 영향을 미치지 않습니다. 레이블에 마크다운을 사용할 수 있습니다.
- `key`(str, int) : 위젯의 고유 키로 사용할 문자열 또는 정수로 요소에 ID를 부여하는데 사용합니다. `key`가 생략된 경우, 위젯의 내용을 기반으로 키가 생성됩니다. `key`는 위젯의 상태를 식별하고 추적하기 때문에 동일한 유형의 여러 위젯은 동일한 키를 공유할 수 없습니다. 만약, 동일 위젯이 동일 `label`을 가지고 버튼을 생성하였을 때, 동일한 키를 공유할 수 없어 DuplicateWidgetID 에러가 발생합니다.
- `help`(str*,* None, Keyword only) : 사용자에게 알약 위젯 레이블 옆에 도움말 텍스트를 제공하기 위해 사용합니다. 지정한 도움말 텍스트는 알약 위젯 레이블 옆에 "?" 아이콘으로 표시되며, 마우스를 가져다 대면 툴팁으로 표시됩니다. label_visibility="visible"일 때만 툴팁을 표시하며, 설명에는 마크다운을 사용할 수 있습니다. None(기본값)이면 툴팁이 표시되지 않습니다.
- `on_change`(callable, Keyword only) : 다중 선택 알약 위젯 값이 변경될 때 호출되는 선택적 콜백 함수입니다.
- `args`(tuple, Keyword only) : 콜백에 전달할 선택적 인수 튜플입니다.
- `kwargs`(dict, Keyword only) : 콜백에 전달할 선택적 `kwargs` 딕셔너리입니다.
- `disabled`(bool, Keyword only) : 알약 위젯을 활성화하거나 비활성화할 수 있습니다.
  - False(기본값) : 알약 위젯 활성화
  - True : 알약 위젯 비활성화, 비활성화된 위젯은 사용자가 어떠한 조작을 입력할 수 없습니다.
- `label_visibility`("visible", "hidden", "collapsed", Keyword only) : 레이블의 표시 여부입니다.
  - "visible"(기본값) : 레이블을 필드 위에 표시합니다. 필드는 위젯의 값이 입력되는 공간을 의미합니다.
  - "hidden" : 레이블은 표시되지 않지만, 위젯 위에 빈 공간(공백)을 표시하여 위젯을 다른 위젯과 정렬하는 데 도움을 줍니다.(label=""와 동일)
  - "collapsed" : 레이블과 공백이 모두 제거됩니다. 기본값은 `label`을 보여 줍니다.

**반환값**

- list of V, V, None
  - selection_mode="multi" : 선택한 옵션의 목록 또는 빈 목록입니다.
  - selection_mode="single" : 선택된 옵션 또는 None입니다.

**예시 1**

다음 예시는 다중 선택 알약 위젯을 표시하고 선택 항목을 표시합니다.

```python

selection = st.pills("Directions", options, selection_mode="multi")

```

## st.segmented_control

`st.segmented_control`는 세그먼트 제어 위젯을 표시합니다. 세그먼트 제어 위젯은 전달된 각 옵션이 토글 버튼처럼 작동하는 선형 세그먼트 집합입니다.

```python
st.segmented_control(label, options, *, selection_mode="single", default=None, format_func=None, key=None, help=None, on_change=None, args=None, kwargs=None, disabled=False, label_visibility="visible")
```

**매개변수**

- `label`(str) : 사용자에게 세그먼트 제어 위젯이 어떤 용도로 사용되는지 간략히 설명하는 레이블입니다. 레이블에는 마크다운을 포함할 수 있습니다. 이미지는 아이콘처럼 표시되며 최대 높이는 글꼴 높이와 같습니다. 지원되지 않는 마크다운 요소는 래핑이 해제되어 해당 하위 요소(텍스트 콘텐츠)만 렌더링됩니다. 지원되지 않는 요소는 백슬래시 이스케이프를 사용하여 리터럴 문자로 표시합니다. ex) 1\. 정렬된 목록이 아님
  접근성을 위해 빈 레이블을 설정해서는 안 되지만, 필요한 경우 `label_visibility`를 사용하여 숨길 수 있습니다. 향후에는 예외를 발생시켜 빈 레이블을 허용하지 않을 수 있습니다.
- `options`(Iterable) : 이터러블의 선택 옵션에 대한 레이블입니다. 선택 가능한 항목들이 list, set, Sequence[V], numpy.ndarray, pandas.Series, pandas.DataFrame, pandas.Index 등으로 나타낼 수 있습니다. 사용자가 리스트에서 여러 개의 항목을 선택할 수 있습니다. 옵션이 데이터프레임과 유사한 경우 첫 번째 열이 사용됩니다. 각 레이블은 기본적으로 내부적으로 str으로 형변환되며, 마크다운을 사용할 수 있습니다.
- `selection_mode`("single", "multi", Keyword only) : 위젯의 선택 모드입니다.
  - "single"(기본값) : 하나의 옵션만 선택할 수 있습니다.
  - "multi" : 여러 옵션을 선택할 수 있습니다.
- `default`(Iterable of V, V, None, Keyword only) : 위젯이 처음 렌더링될 때 위젯의 값입니다.
  - selection_mode="multi" : 리스트, 단일 값 또는 None이 될 수 있습니다.
  - selection_mode="single" : 단일 값 또는 None이 될 수 있습니다.
- `format_func`(function, Keyword only) : 세그먼트 제어 위젯 옵션의 레이블을 변경하기 위한 함수입니다. 원시 옵션을 인수로 받고 해당 옵션에 표시할 레이블을 출력해야 합니다. 이 함수는 명령의 반환 값에는 영향을 미치지 않습니다. 레이블에 마크다운을 사용할 수 있습니다.
- `key`(str, int, Keyword only) : 위젯의 고유 키로 사용할 문자열 또는 정수로 요소에 ID를 부여하는데 사용합니다. `key`가 생략된 경우, 위젯의 내용을 기반으로 키가 생성됩니다. `key`는 위젯의 상태를 식별하고 추적하기 때문에 동일한 유형의 여러 위젯은 동일한 키를 공유할 수 없습니다. 만약, 동일 위젯이 동일 `label`을 가지고 버튼을 생성하였을 때, 동일한 키를 공유할 수 없어 DuplicateWidgetID 에러가 발생합니다.
- `help`(str*,* None, Keyword only) : 사용자에게 세그먼트 제어 위젯 레이블 옆에 도움말 텍스트를 제공하기 위해 사용합니다. 지정한 도움말 텍스트는 세그먼트 제어 위젯 레이블 옆에 "?" 아이콘으로 표시되며, 마우스를 가져다 대면 툴팁으로 표시됩니다. label_visibility="visible"일 때만 툴팁을 표시하며, 설명에는 마크다운을 사용할 수 있습니다. None(기본값)이면 툴팁이 표시되지 않습니다.
- `on_change`(callable, Keyword only) : 세그먼트 제어 위젯 값이 변경될 때 호출되는 선택적 콜백 함수입니다.
- `args`(tuple, Keyword only) : 콜백에 전달할 선택적 인수 튜플입니다.
- `kwargs`(dict, Keyword only) : 콜백에 전달할 선택적 `kwargs` 딕셔너리입니다.
- `disabled`(bool, Keyword only) : 세그먼트 제어 위젯을 활성화하거나 비활성화할 수 있습니다.
  - False(기본값) : 세그먼트 제어 위젯 활성화
  - True : 세그먼트 제어 위젯 비활성화, 비활성화된 위젯은 사용자가 어떠한 조작을 입력할 수 없습니다.
- `label_visibility`("visible", "hidden", "collapsed", Keyword only) : 레이블의 표시 여부입니다.
  - "visible"(기본값) : 레이블을 필드 위에 표시합니다. 필드는 위젯의 값이 입력되는 공간을 의미합니다.
  - "hidden" : 레이블은 표시되지 않지만, 위젯 위에 빈 공간(공백)을 표시하여 위젯을 다른 위젯과 정렬하는 데 도움을 줍니다.(label=""와 동일)
  - "collapsed" : 레이블과 공백이 모두 제거됩니다. 기본값은 `label`을 보여 줍니다.

**반환값**

- list of V, V, None
  - selection_mode="multi" : 선택한 옵션의 목록 또는 빈 목록입니다.
  - selection_mode="single" : 선택된 옵션 또는 None입니다.

**예시 1**

```python

selection = st.segmented_control(
    "Directions", options, selection_mode="multi"
```

```python
st.select_slider(label, options=(), value=None,
```

- `format_func`(function) : 슬라이더 위젯 옵션의 레이블을 변경하기 위한 함수입니다. 이 함수는 옵션을 인자로 받으며, 출력값은 str로 형변환됩니다.
- `key`(str, int) : 위젯의 고유 키로 사용할 문자열 또는 정수로 요소에 ID를 부여하는데 사용합니다. `key`가 생략된 경우, 위젯의 내용을 기반으로 키가 생성됩니다. `key`는 위젯의 상태를 식별하고 추적하기 때문에 동일한 유형의 여러 위젯은 동일한 키를 공유할 수 없습니다. 만약, 동일 위젯이 동일 `label`을 가지고 버튼을 생성하였을 때, 동일한 키를 공유할 수 없어 DuplicateWidgetID 에러가 발생합니다.
- `help`(str*,* None) : 사용자에게 슬라이더 위젯 레이블 옆에 도움말 텍스트를 제공하기 위해 사용합니다. 지정한 도움말 텍스트는 슬라이더 위젯 레이블 옆에 "?" 아이콘으로 표시되며, 마우스를 가져다 대면 툴팁으로 표시됩니다. label_visibility="visible"일 때만 툴팁을 표시하며, 설명에는 마크다운을 사용할 수 있습니다. None(기본값)이면 툴팁이 표시되지 않습니다.
- `on_change`(callable) : 슬라이더 위젯의 값이 변경될 때 호출되는 선택적 콜백 함수입니다.
- `args`(tuple, Keyword only) : 콜백에 전달할 선택적 인수 튜플입니다.
- `kwargs`(dict, Keyword only) : 콜백에 전달할 선택적 `kwargs` 딕셔너리입니다.
- `disabled`(bool) : 슬라이더 위젯을 활성화하거나 비활성화할 수 있습니다.
  - False(기본값) : 슬라이더 위젯 활성화
  - True : 슬라이더 위젯 비활성화, 비활성화된 위젯은 사용자가 어떠한 조작을 입력할 수 없습니다.
- `label_visibility`("visible", "hidden", "collapsed", Keyword only) : 레이블의 표시 여부입니다.
  - "visible"(기본값) : 레이블을 필드 위에 표시합니다. 필드는 위젯의 값이 입력되는 공간을 의미합니다.
  - "hidden" : 레이블은 표시되지 않지만, 위젯 위에 빈 공간(공백)을 표시하여 위젯을 다른 위젯과 정렬하는 데 도움을 줍니다.(label=""와 동일)
  - "collapsed" : 레이블과 공백이 모두 제거됩니다. 기본값은 `label`을 보여 줍니다.

**반환값**

- any value, tuple of any value : 슬라이더 위젯의 현재 값입니다. 반환 유형은 값 매개변수의 데이터 유형과 일치합니다.

## st.slider

`st.slider`는 슬라이더 위젯을 생성하여, 사용자가 숫자 범위 내에서 값 선택 및 연속적인 값을 선택할 수 있습니다. 값은 정수, 실수, 날짜, 시간 및 날짜/시간 유형을 지원하며, 튜플이나 리스트로 값으로 전달하여 범위 슬라이더를 렌더링할 수 있습니다.

+/- (1<<53) - 1을 초과하는 정수 값은 Python 서버와 JavaScript 클라이언트 간의 직렬화 제약 조건으로 인해 위젯에서 정확하게 저장하거나 반환할 수 없습니다. 이러한 숫자는 부동 소수점으로 처리해야 하므로 정밀도가 떨어집니다.

- `st.select_slider` : 모든 데이터 유형을 허용하고 반복 가능한 옵션 집합을 받습니다.
- `st.slider` : 숫자 또는 날짜/시간 데이터만 허용하고 범위를 입력으로 받습니다.

```python
st.slider(label, min_value=None, max_value=None, value=None, step=None,
          format=None, key=None, help=None, on_change=None, args=None,
          kwargs=None, *, disabled=False, label_visibility="visible")
```

**매개변수**

- `label`(str) : 사용자에게 슬라이더 위젯이 어떤 용도로 사용되는지 간략히 설명하는 레이블입니다. 레이블에는 마크다운을 포함할 수 있습니다. 이미지는 아이콘처럼 표시되며 최대 높이는 글꼴 높이와 같습니다. 지원되지 않는 마크다운 요소는 래핑이 해제되어 해당 하위 요소(텍스트 콘텐츠)만 렌더링됩니다. 지원되지 않는 요소는 백슬래시 이스케이프를 사용하여 리터럴 문자로 표시합니다. ex) 1\. 정렬된 목록이 아님
  접근성을 위해 빈 레이블을 설정해서는 안 되지만, 필요한 경우 `label_visibility`를 사용하여 숨길 수 있습니다. 향후에는 예외를 발생시켜 빈 레이블을 허용하지 않을 수 있습니다.
- `min_value`(a supported type, None) : 슬라이더 입력 위젯에 최솟값을 설정할 수 있습니다. 값이 정수인 경우 기본값은 0, 부동 소수점인 경우 0.0, 날짜/날짜/시간인 경우 value - timedelta(days=14), 시간인 경우 time.min입니다.
- `max_value`(a supported type, None) : 슬라이더 입력 위젯에 최댓값을 설정할 수 있습니다. 값이 정수인 경우 기본값은 100, 부동 소수점인 경우 1.0, 날짜/날짜/시간인 경우 value + timedelta(days=14), 시간인 경우 time.max입니다.
- `value`(a supported type, a tuple/list of supported types, None) **:** 슬라이더가 처음 렌더링될 때 슬라이더의 값입니다. 여기에 두 값의 tuple/list가 전달되면 하한과 상한이 있는 범위 슬라이더가 렌더링됩니다. 예를 들어 (1, 10)으로 설정하면 슬라이더의 선택 가능한 범위가 1에서 10 사이가 됩니다. 기본값은 첫 번째 옵션입니다.
- `step`(int, float, timedelta, None) : 슬라이더가 이동하는 간격을 설정합니다. `value`의 타입에 따라 기본값이 다르게 설정됩니다.
  - int(정수)의 기본값은 1
  - float(실수)의 기본값은 0.01
  - date/datetime(날짜, 날짜/시간)의 기본값은 timedelta(days=1)
  - time(시간)의 기본값은 timedelta(minutes=15)(또는 `max_value` - `min_value` < 1 day)
- `format`(str, None) : printf 스타일 포맷 문자열을 지정하여 숫자 표시 형식을 제어합니다. 지정된 표시 형식은 반환값에 영향을 주지 않습니다.
  - int, float 타입은 %d %e %f %g %i 형식을 지원합니다. 정수 및 부동 소수점 서식 지정에 대한 자세한 내용은 [sprintf.js](https://github.com/alexei/sprintf.js?tab=readme-ov-file#format-specification) 를 참조하세요. 예를 들어 format="%0.1f"는 소수점 뒤에 한 자리만 표시되도록 표시되는 소수점 정밀도를 조정합니다.
  - date, time, datetime 타입은 Moment.js 표기법을 사용합니다. Moment.js 표기법에 대한 자세한 사항은 [공식 문서](https://momentjs.com/docs/#/displaying/format/)를 참고 바랍니다. 예를 들어 format="ddd ha"는 요일과 시간("화요일 오후 8시")을 표시하도록 표시되는 날짜 시간을 조정합니다.
- `key`(str, int) : 위젯의 고유 키로 사용할 문자열 또는 정수로 요소에 ID를 부여하는데 사용합니다. `key`가 생략된 경우, 위젯의 내용을 기반으로 키가 생성됩니다. `key`는 위젯의 상태를 식별하고 추적하기 때문에 동일한 유형의 여러 위젯은 동일한 키를 공유할 수 없습니다. 만약, 동일 위젯이 동일 `label`을 가지고 버튼을 생성하였을 때, 동일한 키를 공유할 수 없어 DuplicateWidgetID 에러가 발생합니다.
- `help`(str*,* None) : 사용자에게 슬라이더 위젯 레이블 옆에 도움말 텍스트를 제공하기 위해 사용합니다. 지정한 도움말 텍스트는 슬라이더 위젯 레이블 옆에 "?" 아이콘으로 표시되며, 마우스를 가져다 대면 툴팁으로 표시됩니다. label_visibility="visible"일 때만 툴팁을 표시하며, 설명에는 마크다운을 사용할 수 있습니다. None(기본값)이면 툴팁이 표시되지 않습니다.
- `on_change`(callable) : 슬라이더 위젯 값이 변경될 때 호출되는 선택적 콜백 함수입니다.
- `args`(tuple) : 콜백에 전달할 선택적 인수 튜플입니다.
- `kwargs`(dict) : 콜백에 전달할 선택적 `kwargs` 딕셔너리입니다.
- `disabled`(bool, Keyword only) : 슬라이더를 활성화하거나 비활성화할 수 있습니다.
  - False(기본값) : 슬라이더 위젯 활성화
  - True : 슬라이더 위젯 비활성화, 비활성화된 위젯은 사용자가 어떠한 조작을 입력할 수 없습니다.
- `label_visibility`("visible", "hidden", "collapsed", Keyword only) : 레이블의 표시 여부입니다.
  - "visible"(기본값) : 레이블을 필드 위에 표시합니다. 필드는 위젯의 값이 입력되는 공간을 의미합니다.
  - "hidden" : 레이블은 표시되지 않지만, 위젯 위에 빈 공간(공백)을 표시하여 위젯을 다른 위젯과 정렬하는 데 도움을 줍니다.(label=""와 동일)
  - "collapsed" : 레이블과 공백이 모두 제거됩니다. 기본값은 `label`을 보여 줍니다.

**반환값**

- int/float/date/time/datetime, tuple of int/float/date/time/datetime : 슬라이더 위젯의 현재 값입니다. 반환 유형은 값 매개변수의 데이터 유형과 일치합니다.

```python
st.selectbox(label, options, index=0, format_func=special_internal_function, key=None, help=None, on_change=None, args=None, kwargs=None, *, placeholder="Choose an option", disabled=False, label_visibility="visible")
```

- `label`(str) : 사용자에게 셀렉트 박스 위젯이 어떤 용도로 사용되는지 간략히 설명하는 레이블입니다. 레이블에는 마크다운을 포함할 수 있습니다. 이미지는 아이콘처럼 표시되며 최대 높이는 글꼴 높이와 같습니다. 지원되지 않는 마크다운 요소는 래핑이 해제되어 해당 하위 요소(텍스트 콘텐츠)만 렌더링됩니다. 지원되지 않는 요소는 백슬래시 이스케이프를 사용하여 리터럴 문자로 표시합니다. ex) 1\. 정렬된 목록이 아님
  접근성을 위해 빈 레이블을 설정해서는 안 되지만, 필요한 경우 `label_visibility`를 사용하여 숨길 수 있습니다. 향후에는 예외를 발생시켜 빈 레이블을 허용하지 않을 수 있습니다.
- `options`(Iterable) : 이터러블의 선택 옵션에 대한 레이블입니다. 선택 가능한 항목들이 list, set, Sequence[V], numpy.ndarray, pandas.Series, pandas.DataFrame, pandas.Index 등으로 나타낼 수 있습니다. 사용자가 리스트에서 여러 개의 항목을 선택할 수 있습니다. 옵션이 데이터프레임과 같은 경우 첫 번째 열이 사용됩니다. 각 레이블은 기본적으로 내부적으로 str으로 형변환됩니다.
- `index`(int) : 처음 설정 시 미리 선택된 옵션의 인덱스입니다. None이면 사용자가 옵션을 선택할 때까지 비어 있는 상태로 초기화되고 None을 반환합니다. 기본값은 0(첫 번째 옵션)입니다.
- `format_func`(function) : 셀렉트 박스 옵션의 레이블을 변경하기 위한 함수입니다. 원시 옵션을 인수로 받고 해당 옵션에 표시할 레이블을 출력해야 합니다. 이는 명령의 반환 값에는 영향을 미치지 않습니다.
- `key`(str, int) : 위젯의 고유 키로 사용할 문자열 또는 정수로 요소에 ID를 부여하는데 사용합니다. `key`가 생략된 경우, 위젯의 내용을 기반으로 키가 생성됩니다. `key`는 위젯의 상태를 식별하고 추적하기 때문에 동일한 유형의 여러 위젯은 동일한 키를 공유할 수 없습니다. 만약, 동일 위젯이 동일 `label`을 가지고 버튼을 생성하였을 때, 동일한 키를 공유할 수 없어 DuplicateWidgetID 에러가 발생합니다.
- `help`(str*,* None) : 사용자에게 셀렉트 박스 위젯 레이블 옆에 도움말 텍스트를 제공하기 위해 사용합니다. 지정한 도움말 텍스트는 셀렉트 박스 위젯 레이블 옆에 "?" 아이콘으로 표시되며, 마우스를 가져다 대면 툴팁으로 표시됩니다. label_visibility="visible"일 때만 툴팁을 표시하며, 설명에는 마크다운을 사용할 수 있습니다. None(기본값)이면 툴팁이 표시되지 않습니다.
- `on_change`(callable) : 셀렉트 박스의 값이 변경될 때 호출되는 선택적 콜백 함수입니다.
- `args`(tuple) : 콜백에 전달할 선택적 인수 튜플입니다.
- `kwargs`(dict) : 콜백에 전달할 선택적 `kwargs` 딕셔너리입니다.
- `placeholder`(str, Keyword only) : 선택된 옵션이 없을 때 표시할 문자열입니다. 기본값은 "옵션 선택"입니다.
- `disabled`(bool, Keyword only) : 셀렉트 박스 위젯을 활성화하거나 비활성화할 수 있습니다.
  - False(기본값) : 셀렉트 박스 위젯 활성화
  - True : 셀렉트 박스 위젯 비활성화, 비활성화된 위젯은 사용자가 어떠한 조작을 입력할 수 없습니다.
- `label_visibility`("visible", "hidden", "collapsed", Keyword only) : 레이블의 표시 여부입니다.
  - "visible"(기본값) : 레이블을 필드 위에 표시합니다. 필드는 위젯의 값이 입력되는 공간을 의미합니다.
  - "hidden" : 레이블은 표시되지 않지만, 위젯 위에 빈 공간(공백)을 표시하여 위젯을 다른 위젯과 정렬하는 데 도움을 줍니다.(label=""와 동일)
  - "collapsed" : 레이블과 공백이 모두 제거됩니다. 기본값은 `label`을 보여 줍니다.

**반환값**

- any : 선택한 옵션 또는 선택된 옵션이 없는 경우 None입니다.

**예시 1**

다음 예시는 드롭다운 형식으로 목록에서 특정 항목을 선택합니다. 사용자가 드롭다운 목록에서 선택하면 위와 같이 사용자가 선택한 항목이 출력됩니다.

```python
import streamlit as st

sports = ["야구","축구","농구","배드민턴"]

selected_sports = st.selectbox("좋아하는 운동을 선택하세요", sports)

if selected_sports:
    st.write(f"좋아하는 운동 : {selected_sports}")
```

**예시 2**

다음 예시는 `index`를 None으로 주어 빈 선택 상자를 초기화하고 `placeholder`에 “운동 종목을 선택하세요...”라고 작성하여 좋아하는 운동을 선택할 수 있도록 유도합니다.

```python
import streamlit as st

sports = ("야구","축구","농구","배드민턴")

selected_sports = st.selectbox("좋아하는 운동을 선택하세요", sports, index=None, placeholder="운동 종목을 선택하세요...")

```

**예시 3**

다음 예시는 `disabled`를 사용하여 위젯을 비활성화하거나 `label_visibility`를 사용하여 서로 다른 라벨 출력 여부를 설정합니다.

```python
import streamlit as st

disabled = st.checkbox("셀렉트 박스 위젯 비활성화")
visibility = st.radio(
    "라벨을 설정해보세요 👇",
    options=["visible", "hidden", "collapsed"],
)
sports = ("야구","축구","농구","배드민턴")
option = st.selectbox(
    "좋아하는 운동을 선택하세요", sports,
    label_visibility=visibility,
    disabled=disabled)
st.write(f"당신이 좋아하는 운동은 {option}입니다!")
```

## st.toggle

`st.toggle`는 토글 위젯을 생성합니다.

```python
st.toggle(label, value=False, key=None, help=None, on_change=None, args=None, kwargs=None, *, disabled=False, label_visibility="visible")
```

**매개변수**

- `label`(str) : 사용자에게 토글이 어떤 용도로 사용되는지 간략히 설명하는 레이블입니다. 레이블에는 마크다운을 포함할 수 있습니다. 이미지는 아이콘처럼 표시되며 최대 높이는 글꼴 높이와 같습니다. 지원되지 않는 마크다운 요소는 래핑이 해제되어 해당 하위 요소(텍스트 콘텐츠)만 렌더링됩니다. 지원되지 않는 요소는 백슬래시 이스케이프를 사용하여 리터럴 문자로 표시합니다. ex) 1\. 정렬된 목록이 아님
  접근성을 위해 빈 레이블을 설정해서는 안 되지만, 필요한 경우 `label_visibility`를 사용하여 숨길 수 있습니다. 향후에는 예외를 발생시켜 빈 레이블을 허용하지 않을 수 있습니다.
- `value`(bool) : 토글의 초기 상태를 보여주는 bool 값입니다.
  - True : 토글이 on 되어있는 상태입니다.
  - False(기본값) : 토글이 off 되어있는 상태입니다.
- `key`(str, int) : 위젯의 고유 키로 사용할 문자열 또는 정수로 요소에 ID를 부여하는데 사용합니다. `key`가 생략된 경우, 위젯의 내용을 기반으로 키가 생성됩니다. `key`는 위젯의 상태를 식별하고 추적하기 때문에 동일한 유형의 여러 위젯은 동일한 키를 공유할 수 없습니다. 만약, 동일 위젯이 동일 `label`을 가지고 버튼을 생성하였을 때, 동일한 키를 공유할 수 없어 DuplicateWidgetID 에러가 발생합니다.
- `help`(str*,* None) : 사용자에게 토글 위젯 레이블 옆에 도움말 텍스트를 제공하기 위해 사용합니다. 지정한 도움말 텍스트는 토글 위젯 레이블 옆에 "?" 아이콘으로 표시되며, 마우스를 가져다 대면 툴팁으로 표시됩니다. label_visibility="visible"일 때만 툴팁을 표시하며, 설명에는 마크다운을 사용할 수 있습니다. None(기본값)이면 툴팁이 표시되지 않습니다.
- `on_change`(callable) : 토글 값이 변경될 때 호출되는 선택적 콜백 함수입니다.
- `args`(tuple, Keyword only) : 콜백에 전달할 선택적 인수 튜플입니다.
- `kwargs`(dict, Keyword only) : 콜백에 전달할 선택적 `kwargs` 딕셔너리입니다.
- `disabled`(bool) : 토글을 활성화하거나 비활성화할 수 있습니다.
  - False(기본값) : 토글 활성화
  - True : 토글 비활성화, 비활성화된 위젯은 사용자가 어떠한 조작을 입력할 수 없습니다.
- `label_visibility`("visible", "hidden", "collapsed", Keyword only) : 레이블의 표시 여부입니다.
  - "visible"(기본값) : 레이블을 필드 위에 표시합니다. 필드는 위젯의 값이 입력되는 공간을 의미합니다.
  - "hidden" : 레이블은 표시되지 않지만, 위젯 위에 빈 공간(공백)을 표시하여 위젯을 다른 위젯과 정렬하는 데 도움을 줍니다.(label=""와 동일)
  - "collapsed" : 레이블과 공백이 모두 제거됩니다. 기본값은 `label`을 보여 줍니다.

**반환값**

- bool : 토글이 선택되어 있는지 여부입니다.
