# JSX 좀 더 알아보기

우리는 React를 사용하기 위해 JSX를 사용합니다. 그럼 JSX가 없다면 어떻게 React Element와 컴포넌트를 만들까요?

## `JSX` 없이 React사용해보기

JSX없이 순수하게 React만을 사용한다면 아래와 같이 만듭니다.

```jsx
const reactElement = React.createElement(
		"div",
		{className:'simple-class'},
		"테스트"
)
```

## `JSX`로 만든 React요소

이제 앞서 만든 React element를 로그로 출력해보겠습니다.

```jsx
const reactElement = React.createElement(
		"div",
		{className:'simple-class'},
		"테스트"
)
console.log(reactElement )
```

## 비교

이번에는 `jsx`를 통해 React Element를 만들고 출력해보겠습니다.

```jsx
const reactElement = <div className='simple-class'>테스트</div>
console.log(reactElement)
```

```jsx
const reactElement1 = React.createElement(
		"div",

console.log("jsx를 사용하지않은 컴포넌트 : "reactComponent1)
```

만들어낸 두 요소는 거의 똑같은 모습을 하고 있습니다.

className과 태그 안에 넣어둔 children이 props에 들어간 모습도 볼 수있구요. 우리가 태그에 넣은 여러값들은 props에 객체 형태로 들어갑니다. 컴포넌트를 만들 때 `props.children`과 같은 형식으로 사용할 수 있는 이유도 내부적으로 위와 같이 변경해주기 때문입니다.

## props.children

만약 아래와 같은 경우 어떻게 만들어야할까요?

```jsx
return(
	<div>
		<div>
			<div>
				<div></div>
			</div>
		</div>
	</div>
)
```

아래와 같이 만드시면 됩니다!

```jsx
const reactElement1 = React.createElement(
		"div",
		{className:'simple-class'},
		React.createElement(
            "div",
            {className:'simple-class'},
            React.createElement(
                "div",
                {className:'simple-class'},
                React.createElement(
                    "div",
                    {className:'simple-class'},
                    
                )
            )
        )
    )
    console.log("jsx를 사용하지않은 컴포넌트 : ",reactElement1)
```

**createElement**에서 3번째 입력값이 `props.children`으로 들어가는 것을 확인했습니다. 또, jsx에서 태그 안에 들어가는 내용들도 `props.chlidren`으로 들어가고 있었죠.

따라서 위와 같이 작성을 할 수 있습니다. 마찬가지로 위와 같이 만든 컴포넌트를 로그로 출력해보면 아래와 같이 출력됩니다.

## Props

결국 props는 → `리액트 컴포넌트를 사용하며 정의한 어트리뷰트들` ←이 모여있는 객체입니다.

![컴포넌트를 사용했을때  출력되는 props의 모습](Untitled%2080.png)

컴포넌트를 사용했을때  출력되는 props의 모습

이벤트를 주기 위해 넣은 `onClick`과 같은 `이벤트`들도 마찬가지로 props로 들어가게 되고, `props`에 컴포넌트 또한 넣어서 사용할 수 있겠죠! 컴포넌트를 만들고 `props`를 `console.log`를 통해 확인해보는 것도 좋을 것 같습니다.

## 결론

즉, **JSX는 React사용을 편하게 해주는 문법입니다.**
