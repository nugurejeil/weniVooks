# 백업

# 3. 사용자 정의 슬래시 명령어

claude code에서 기본적으로 제공하는 명령어 외에도 자주 사용하는 프롬프트나 작업 과정을 마크다운으로 작성하여 명령어로 사용할 수 있습니다. 코드에 설명을 주석으로 추가하거나 작업 내용을 요약하는 작업 등 요청을 반복하여 하고 있다면 간단한 명령어로 작성하여 작업 과정을 효율적으로 관리할 수 있습니다.

<aside>
💡

사용자 정의 슬래시 명령어는 프로젝트 명령어와 개인 명령어로 구분되지만, 저장 위치에 따라 구분되는 것일 뿐 작성하는 방식은 동일합니다.

프로젝트 명령어는 `.claude/commands/` 폴더 안에 작성하여 해당 프로젝트 내에서 팀과 공유하며 사용할 수 있고, 개인 명령어는 `~/.claude/commands` 폴더와 같이 최상위 폴더에서 작성되어 모든 프로젝트에서 가져와 개인적으로 사용할 수 있습니다.

</aside>

`.claude` 폴더 안에 `commands` 폴더를 새로 만들어주겠습니다. 만약 작업하고 있는 프로젝트 폴더에 `.claude` 폴더가 없다면 새로 만들어주세요. `commands` 폴더 안에 내가 사용할 명령어 이름으로 마크다운 파일을 생성합니다. comment, task-summary 등 작업을 잘 나타낼 수 있는 이름을 사용하여 파일명을 설정합니다. 이때, claude에서 제공하는 기본 명령어와 같은 파일명을 사용하면 우리가 작성한 명령어가 기본 명령어를 덮어쓰기 때문에 주의해야 합니다.

```bash
---
description: 지정된 파일에 상세한 주석과 설명 추가
argument-hint: [파일경로] 
---
```
